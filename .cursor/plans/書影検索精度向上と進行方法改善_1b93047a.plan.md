# 書影検索の精度向上と進行方法改善計画

## 現状分析

会話履歴から以下の課題が確認されました：

### 精度の問題

1. **テキストベース検索の限界**

- 視覚的特徴（色、構図、スタイル）を言葉で正確に伝えられない
- キーワード検索では類似表紙を見つけにくい
- 記憶が曖昧で、条件が徐々に変化する

2. **検索範囲の広さ**

- 出版社、年代、ジャンルが不明確
- 候補が多すぎて絞り込めない
- 類似表紙が複数存在する

### 進行方法の問題

1. **非体系的な検索**

- 検索が散漫で、優先順位が不明確
- 同じ候補を何度も検討している
- フィードバックが非効率

2. **情報収集の非効率性**

- 一度に多くの質問をして混乱を招く
- 重要な情報と瑣末な情報の区別がついていない

## 改善提案

### 1. 構造化された情報収集プロセス

#### Phase 1: 基本情報の確定（必須項目）

- **視覚的特徴の構造化**
- 人物の特徴（シルエット/実写、ポーズ、性別、人数）
- 背景の特徴（色、スタイル、要素）
- レイアウト（人物と背景の配置関係）
- スタイル（写実的/抽象的、アート的/商業的）

- **書籍情報の確定**
- 刊行時期（見た時期、出版時期の推定）
- 出版社・レーベル（背表紙の色、ロゴの記憶）
- 判型（文庫/単行本/新書）
- ジャンル（SF/ファンタジー/文芸など）

#### Phase 2: 類似表紙からの逆引き

- **参照表紙の活用**
- 「近い」と感じた表紙を複数提示してもらう
- 各参照表紙との「違い」を明確化
- 参照表紙の共通要素を抽出

#### Phase 3: 段階的絞り込み

- **優先順位の明確化**

1. 出版社・レーベルで絞り込み
2. 刊行時期で絞り込み
3. 表紙デザイナー・イラストレーターで絞り込み
4. タイトルキーワードで絞り込み

### 2. 画像検索の活用戦略

#### 推奨ツールと使い分け

1. **Google画像検索（逆引き検索）**

- 参照表紙画像をアップロードして類似画像を検索
- 「この画像で検索」機能を活用

2. **Pinterest**

- 視覚的キーワードで検索
- 類似表紙のコレクションを参照

3. **書籍データベース**

- 国立国会図書館サーチ
- 各出版社の公式サイト
- 古書店サイト（ブックオフ、日本の古本屋など）

#### 検索キーワードの体系化

- **視覚的特徴キーワード**
- 「SF 表紙 シルエット 人物 惑星」
- 「ハヤカワ文庫SF 表紙 宇宙 人物」
- 「創元SF文庫 表紙 イラスト 宇宙」

- **デザイナー・イラストレーターキーワード**
- 「生頼範義 SF 表紙」
- 「加藤直之 ハヤカワ 表紙」
- 「土井宏明 フィリップ・K・ディック」

### 3. 効率的なフィードバックループ

#### 候補提示の方法

- **3-5件に絞り込んで提示**
- 多すぎると判断が困難
- 各候補に「近い点」「違う点」を明記

#### フィードバックの構造化

- **「近い」と感じた候補について**
- どの要素が近いか（色/構図/スタイル）
- どの要素が違うか（具体的に）
- 近さの度合い（10段階評価）

- **「違う」と感じた候補について**
- なぜ違うと感じるか（1-2点に絞る）
- この候補から学べることはあるか

### 4. 代替アプローチの提案

#### タイトル・内容からの逆引き

- 表紙が特定できない場合の代替手段
- 内容の断片的な記憶から検索
- 推薦者（成毛眞など）のリストから検索

#### コミュニティ活用

- Yahoo!知恵袋、Reddit、Twitterなど
- 専門コミュニティ（SF読書会、書評サイト）
- 図書館司書への相談

## 実装推奨事項

### 1. チェックリスト形式の情報収集

構造化された質問フォームを作成し、段階的に情報を収集

### 2. 参照表紙の比較表

複数の参照表紙を並べて、違いを視覚的に比較

### 3. 検索履歴の記録

検討した候補とその結果を記録し、重複を避ける

### 4. 画像検索の優先活用

テキスト検索より画像検索を優先し、視覚的マッチングを重視

## 次のステップ

1. 現在の情報を構造化して整理
2. 参照表紙（「永遠の終り」「タウ・ゼロ」など）を画像検索で逆引き
3. 出版社・レーベルを確定させる質問を優先
4. 候補を3-5件に絞り込んで提示
5. フィードバックを構造化して収集
