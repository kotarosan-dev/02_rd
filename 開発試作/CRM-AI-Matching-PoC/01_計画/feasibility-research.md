## 1. 調査の背景

CRMデータとAIを組み合わせた人材紹介マッチングシステムの実現可能性について調査。

## 3. 推奨アーキテクチャ

### 3.1 システム構成図

```
┌─────────────────────────────────────────────────────────────────┐
│                        Zoho CRM                                  │
│  ┌──────────────┐    ┌──────────────────────────────────────┐  │
│  │  求職者タブ   │    │  カスタムウィジェット                 │  │
│  │  求人タブ     │───▶│  「おすすめマッチング」表示            │  │
│  └──────────────┘    └──────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼ API呼び出し
┌─────────────────────────────────────────────────────────────────┐
│                    Zoho Catalyst (Serverless)                    │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  Python Function                                          │  │

│  │  - Pinecone検索                                           │  │
│  │  - マッチングスコア計算                                    │  │
│  └──────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
          │                              │
          ▼                              ▼
┌─────────────────────┐      ┌─────────────────────┐
│    OpenAI API       │      │     Pinecone        │
│  (Embeddings生成)   │      │  (ベクトルDB検索)   │
```

### 3.2 データフロー

1. **データ登録時（バッチ処理）**
   - CRMに求職者/求人データが登録される
   - Cron Functionで定期的にデータを取得
   - OpenAI Embeddings APIでベクトル化
   - Pineconeにベクトルデータを保存

2. **マッチング表示時（リアルタイム）**
   - ユーザーが求職者/求人レコードを開く
   - ウィジェットがCatalyst Functionを呼び出し
   - 当該レコードのベクトルでPinecone検索
   - 類似度スコア付きでマッチング候補を返却

## 6. 技術オプション比較

### 6.1 ベクトルデータベース

| サービス | 無料枠 | 有料プラン | 特徴 |
|---------|--------|-----------|------|
| **Pinecone** | 100万ベクトル | $50/月〜 | 業界標準、高速 |
| Weaviate | 一定制限あり | 従量課金 | オープンソース |
| Qdrant | セルフホスト無料 | クラウド版有料 | オンプレ対応 |
| Zoho CRM内蔵 | Zia機能 | エディションに依存 | 追加コストなし |

### 6.2 Zoho Zia 内蔵機能（要検討）

Zoho CRM標準のAI機能「Zia」に以下の機能が存在：

- **Recommendation Builder**: 顧客への製品/サービス推薦
- **Similarity Recommender**: 類似プロファイルのマッチング
- **Next-Best-Experience**: 最適なアクション提案

**検討ポイント**: カスタム開発前に、Ziaの標準機能で要件を満たせるか確認する価値あり。

## 9. 結論と推奨アクション

### 結論
**技術的・コスト的に十分実現可能**。日本の大手人材会社でも類似システムを導入済みであり、技術的な不確実性は低い。

### 推奨アクション

1. **短期（今週中）**: Zoho Ziaの標準機能（Recommendation Builder等）で要件を満たせるか確認
2. **短期（2週間以内）**: カスタム開発が必要な場合、PoC開発を実施
3. **中期（1ヶ月）**: PoC結果を踏まえ、本番導入の判断

### 提案のハードルを下げるには

ミーティングで「提案のハードルが高い」との懸念があったが、以下のアプローチで軽減可能：

- **Zoho Zia標準機能の活用提案**: 追加開発なしで試せる
- **スモールスタート**: まず小規模（100件程度）でPoC実施
- **明確なROI提示**: Adeccoの「年間8万時間削減」事例を引用
