# Zoho CRM Three.js Widget Demo Playbook

Place this in `<ProjectRoot>/01_計画/project_management_docs/zoho_crm_threejs_widget_demo_playbook.md`.

## デモの言い方

悪い言い方:

- Three.js で3Dダッシュボードが作れます
- CRMをかっこよくできます
- リッチなUIにできます

良い言い方:

- Zoho CRM の中で、商談レビューや引き継ぎの判断材料を1画面に集約できます
- 既存の関連リストやレポートでは見えにくい偏りを、担当者がすぐ見つけられるようにします
- CRM から出ずに確認・判断・次アクションまで進めるための小型 widget です

## 10分デモ構成

| 時間 | 内容 | 目的 |
| --- | --- | --- |
| 0:00-1:00 | 「CRMで見たいけど見えない情報」を確認 | 課題を先に置く |
| 1:00-3:00 | Deals Orbit を見せる | 商談レビュー仮説を提示 |
| 3:00-5:00 | Forecast Grid を見せる | 確度別の意思決定に寄せる |

| 9:00-10:00 | 次回は1画面だけ実データで試す提案をする | pilot に進める |

## ヒアリング質問

### 現状把握

- Zoho CRM のどの画面を一番よく見ていますか。
- その画面を見ても判断できず、別資料に移している情報は何ですか。
- 会議や引き継ぎの前に、毎回確認している項目は何ですか。
- 見落としや確認漏れが起きやすい場面はどこですか。

### デモ反応

- 今の5つのデモの中で、自社に一番近いのはどれですか。
- これは誰が、いつ、どの画面で見ると役に立ちそうですか。
- 逆に、3Dである必要がないと感じた部分はどこですか。
- 自社の項目で置き換えるなら、最初に表示したい上位5項目は何ですか。

### 商談化

- これを試すなら、Sandbox と本番のどちらで始めるのが現実的ですか。
- 1画面だけ pilot する場合、どの配置先がよいですか。
- 決裁者はこの画面の何を見れば価値を判断できますか。
- いつまでに判断できる状態にしたいですか。

## BANTCH確認

## 失注・不採用のサイン

- 「かっこいいですね」で終わり、業務画面や会議名が出ない
- 表示したい項目が決まらない
- Zoho標準レポートで足りる
- 決裁者やCRM管理者に接続できない
- データ品質が低すぎて、可視化より先に入力ルール整備が必要

## Pilot 提案文

まず1画面だけ、御社の Zoho CRM 項目で置き換えて試すのが良いです。範囲は `Deals` の商談レビュー画面に絞り、Sandbox か限定ユーザー向けの widget として、実データで「会議前の確認が短くなるか」を見ます。結果が弱ければそこで止め、強ければ Account や Lead に横展開します。
