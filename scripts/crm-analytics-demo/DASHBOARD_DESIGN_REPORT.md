## 2. 構築可能なダッシュボード設計

### ダッシュボード A: **セールスパイプライン可視化**
> 「今月・今四半期の受注予測を一目で把握する」

| チャート | 種類 | データ | 目的 |
|---------|------|--------|------|
| パイプライン合計金額 | KPIカード | SUM(Amount) | 全体規模把握 |
| ステージ別件数・金額 | 棒グラフ（積上げ） | Stage × Amount | ボトルネック発見 |
| 加重受注予測 | KPIカード | SUM(Amount × Probability / 100) | 収益予測 |
| 成約確率別分布 | 円グラフ | Probability グループ | リスク可視化 |
| クロージング月別商談 | 折れ線グラフ | Closing_Month × Amount | 時系列予測 |

**狙いKPI**:
- 加重パイプライン金額: ¥1,425万（合計¥2,730万 × 加重平均52%）
- ステージ進行率（条件確認→受注）

---

### ダッシュボード B: **リード分析・転換率トラッキング**
> 「どのチャネルから質の高いリードが来るか？」

| チャート | 種類 | データ | 目的 |
|---------|------|--------|------|
| リードソース別件数 | 棒グラフ | Lead_Source × COUNT | 流入チャネル評価 |
| ステータス別比率 | 円グラフ | Lead_Status × COUNT | 転換率把握 |
| 業種別年間売上規模 | 棒グラフ | Industry × Annual_Revenue | ターゲット評価 |
| 従業員規模分布 | ヒストグラム | No_of_Employees | 企業規模セグメント |
| リード獲得コスト試算 | KPIカード | Qualified / Total × 100 | 質的指標 |

**狙いKPI**:
- リード質スコア（Qualified率）: 30%（10件中3件）
- 優先度チャネル: Web Site（最多流入）

## 3. KPI → 行動変容フレームワーク

### 「数字が行動を変える」設計思想

```
現状可視化
  ↓
ギャップ認識（目標 vs 実績）
  ↓
行動特定（何が不足？）
  ↓
日次チェック（ダッシュボード確認習慣）
  ↓
行動変容（具体的アクション）
  ↓
数字への反映（1〜2週間後）
```

### 役割別ダッシュボード活用シナリオ

#### 営業担当者
- **見るKPI**: 自分の商談ステージ分布・停滞中の商談
- **促す行動**: 「条件確認フェーズに3週間以上いる商談に今週中にコンタクト」
- **測定方法**: 週次のステージ前進率

#### 営業マネージャー
- **見るKPI**: チーム全体の加重パイプライン・個人別受注率
- **促す行動**: 「加重パイプラインが目標の120%を下回ったら新規開拓活動を強化」
- **測定方法**: 月次パイプライン充填率

#### 経営層
- **見るKPI**: 四半期収益予測・リードソース投資効率
- **促す行動**: 「ROIの高い展示会チャネルに次期予算の30%を集中投資」
- **測定方法**: 四半期受注額・顧客獲得コスト

## 4. クライアント提案時の活用シナリオ

### 提案のストーリーライン

1. **現状の痛み**: 「感覚ベースの商談管理 → 受注見通しが立てられない」
2. **可視化の価値**: 「Analytics で全商談の加重予測を自動計算 → 翌月の受注予測が±15%精度で立つ」
3. **行動変容**: 「ダッシュボードを週次 MTG の起点に → 商談停滞の早期発見率 40%向上」
4. **ROI**: 「月次 1 日分の工数削減 + 受注率 5pt 向上 → 年間¥XXX 万の価値創出」

### 再現性の確認ポイント（本テストで検証済み）

| 確認項目 | 結果 |
|---------|------|
| CRM Deals API での商談作成 | ✅ 8件成功（JP Stage値 + Pipeline指定） |
| CRM Leads API でのリード作成 | ✅ 10件成功（前セッション） |
| Analytics Workspace API作成 | ✅ ID: 102896000000028001 |
| Analytics テーブル作成 | ✅ CRM_Deals_Pipeline + CRM_Leads_Overview |
| Analytics CSV インポート | ✅ Deals 8行 + Leads 10行 |
| API認証（Client Credentials） | ✅ CRM / Analytics 両方成功 |

### 発見した実装上の注意点（スキル更新予定）

1. **Stage値はJP環境では日本語display_value**が必要（`'Qualification'` ではなく `'条件確認'`）
2. **Pipeline は system_mandatory**（`-None-` は使えない、`Standard (Standard)` を明示的に指定）
3. **Analytics ORG ID は CRM ORG ID と別値**（`/restapi/v2/orgs` で取得が必要）
4. **Analytics のCONFIG**: Workspace作成はPOSTボディ、データインポートはURLクエリパラメータ
5. **Analytics データインポート**: `FILE`/`DATA` はmultipart形式、CONFIGキーは小文字（`importType`等）

## 5. 次のステップ（クライアント提案に向けて）
