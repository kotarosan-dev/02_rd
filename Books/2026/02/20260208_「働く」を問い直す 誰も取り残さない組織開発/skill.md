# 脱・能力主義の組織開発フレームワーク

## 概要

「能力」は個人の属性ではなく、環境との相互作用で発現する「状態」である。一元的な能力主義（椅子取りゲーム）から脱却し、個人の「持ち味」を組み合わせることで組織を機能させるアプローチ。

## 使用場面（When to Use）

- 職場がギスギスして人が辞めていく状況の立て直し
- 「優秀な人を採れば解決する」という思考から脱却したいとき
- 新卒・中途社員の定着率を改善したいとき
- マネジャーのリーダーシップスタイルを多様化させたいとき
- ジョブ型人材マネジメントへの移行を検討するとき
- ダイバーシティ推進が数値目標だけで空回りしているとき

## 診断：ギスギス職場の共通パターン

以下の兆候があれば「古い能力主義」の罠に陥っている可能性が高い：

1. **「自分はちゃんとやっている」の蔓延** - 全員が正しいと思い、他者を承認できない
2. **犯人探し思考** - 「マネジャーの力量の問題」「メンバーが優秀じゃない」
3. **スーパーマン人材の要求** - 「即戦力で万能選手で主体的でリーダーシップがあって協調性もある人」
4. **努力の空回り** - 全員が頑張っているのに報われない、徒労感と疲労感
5. **レッテル貼り** - 「働かないおじさん」「言われたことしかやらない若手」

## 4つの実践ツール

### ツール1: チームビルディング・ワークショップ

**目的**: 言いづらかったことを全員で共有し、進むべき方向を見出す

**手順**:
1. 部署全員が参加（管理職も一メンバーとして）
2. 各自の思いを吐き出す場を設定
3. 「言いづらかったこと」も含めて安全に共有
4. レッテル（「働かないおじさん」等）の裏にある本音を発見
5. チームとしての方向性を全員で合意

**ポイント**: 犯人探しでも聖人探しでもない。「誰が悪いか」ではなく「何が起きているか」にフォーカス。

### ツール2: 持ち味の理解と組み合わせ

**目的**: 一元的な「能力」評価から、多元的な「持ち味」の組み合わせへ

**フレームワーク（車の比喩）**:
- **アクセルタイプ**: 推進力がある、行動が速い
- **ブレーキタイプ**: 慎重、リスクを見抜く
- どちらも必要。性別・年齢との相関はない

**手順**:
1. 部署で必要な「機能」を言語化する
2. メンバーそれぞれの持ち味を把握する
3. 持ち味の組み合わせで機能を分担する
4. 「ありもしない個人の能力の見極め」に労力を割かない

### ツール3: オンボーディング

**目的**: 新卒・中途社員を「お手並み拝見」ではなく、計画的に戦力化する

**手順**:
1. 採用段階から「相性」を考慮（内定前から開始）
2. OJT担当者・直属上司含む3名程度でミーティング
3. OJTで起こりうる事態を想定して綿密に準備
4. 繰り返しミーティングでフォロー
5. 「中途社員は使えない」「配属ガチャ」を構造的に防止

### ツール4: モードを選ぶリーダーシップ

**目的**: マネジャーが「こうでなければならない」を捨て、部下に合わせる

**手順**:
1. マネジャー自身の思い込みを自覚する
2. 部下それぞれの持ち味に合わせてリーダーシップスタイルを変える
3. 「メンバーへのケア」を評価項目に組み込む
4. 強制ではなく、対話を通じて自発的な気づきを促す

**重要**: 評価制度に組み込まなければ長続きしない。「マネジャーの心がけ」任せにしない。

## 構造的変革：ジョブ型への移行

4つのツールに加え、より根本的な変革として：

1. **職務要件の明確化** - 曖昧な「能力」ではなく、具体的なジョブを定義
2. **頻繁なフィードバック** - 年1-2回ではなく、プロジェクト途中で小刻みに
3. **キャリアパスの可視化** - 何をどう伸ばせばよいかを明示
4. **「成果責任」の明文化** - 課業（年々変わる）ではなく、達成すべき責任を定義

## 注意点

- 脱・能力主義は「能力が全く不要」ではない。若手育成など「できないことをできるように」するフェーズは存在する
- トップダウンの制度変革とボトムアップの実践の両輪が必要
- 部署単位で始められるが、評価制度への組み込みなしには長続きしない
- 「競争をやめたら国力が下がる」への反論：競争を激化させた30年で生産性も競争力も向上していない事実
