# アカデミック・ライティング・フレームワーク

## 概要

小熊英二『基礎からわかる 論文の書き方』に基づく、論文・レポート・報告書の設計・執筆フレームワーク。「何を書くか」の前に「論文とは何か」を理解し、主題・対象・方法論の三要素を整合させたうえで、パラグラフ・ライティングの原則に従って論理的な文章を構成する。

## 使用場面（When to Use）

- 卒業論文・修士論文・博士論文の計画・執筆
- 大学のレポート課題
- ビジネスにおける報告書・企画書・提案書の作成
- 研究計画書の作成
- 学会発表・プレゼンテーションの準備
- 根拠に基づいた説得的な文章が求められるあらゆる場面

## フレームワーク/プロセス

### Phase 1: 研究設計（主題-対象-方法論の三角形）

```
        主題（Question）
       ／        ＼
      ／  整合性   ＼
     ／            ＼
対象（Object） ── 方法論（Methodology）
```

1. **主題（リサーチ・クエスチョン）**: 抽象的な「問い」の形で設定する
   - 「なぜXはYなのか？」「XはYにどう影響するか？」
   - 「主題は抽象的な問い、対象は具体的に観測できるもの」を厳守
2. **対象**: 具体的に調べられるものに限定する
   - 文書、データ、人物、組織、地域など
3. **方法論（調査設計）**: 主題と対象に適した方法の組み合わせ
   - 量的調査／質的調査／混合法
   - 仮説検証型／仮説生成型

### Phase 2: 先行研究の検討

先行研究を以下の4方向から検討する:
1. **主題の先行研究**: 同じ問いに取り組んだ研究
2. **対象の先行研究**: 同じ対象を扱った研究
3. **方法論の先行研究**: 同じ方法を使った研究
4. **理論的フレームの先行研究**: 依拠する理論体系の研究

### Phase 3: 論文の構成設計

**基本型（ハンバーガー・エッセイ + IMRAD混合）**:
1. 序論: 主題提起、先行研究の検討、対象・方法論の提示
2. 本論: 調査結果の分析と考察（複数章に分割可）
3. 結論: 主題への回答、研究の貢献と限界

### Phase 4: パラグラフ・ライティングによる執筆

原則:
- **一パラグラフに一内容**
- **冒頭にトピック・センテンス**（そのパラグラフで何を述べるか）
- **パラグラフ間の論理的接続**を意識する
- すべてのパラグラフが「主題の論証」に貢献する

### Phase 5: 注記・要約・校正

1. レファレンスの完備（出典の追検証可能性を確保）
2. 要約の作成（各章の縮約 or 主題前面型）
3. 校正チェック（構成/論理、引用/事実、文章/誤字、様式準拠）

## 具体的な実行手順

### ステップ1: 主題と対象の分離
- 自分の関心を「問い」の形に変換する
- 問いに答えるために「具体的に調べるもの」を特定する

### ステップ2: 学問体系の選択
- 同じ対象でも学問体系が変われば主題・方法論が変わる
- 自分の関心に最も適した学問体系を選ぶ

### ステップ3: 方法論の設計
- 方法の「レシピ」を考える（方法の順序と組み合わせ）
- 因果推論の「ミルの三条件」を意識する:
  - 事象Aと事象Bの関連
  - 事象Aの時間的先行
  - A以外の原因の排除

### ステップ4: 構成表の作成
- 論文全体の構成を、章・節・パラグラフの階層で設計する
- 各パラグラフの役割（記述、比較、因果分析など）を明確にする

### ステップ5: 草稿執筆と推敲
- 構成表に従って一パラグラフずつ書く
- 全体を通して論理の流れを確認する

### ステップ6: 最終チェック
- 審査基準に照らした自己点検:
  - 主題・対象・方法は整合しているか
  - 先行研究の検討は十分か
  - 論拠と論理による論証がなされているか
  - 主題に即した結論が導かれているか
  - 学問的貢献が明示されているか

## 入力/出力

### 入力
- 漠然とした関心・問題意識
- 調査・実験・文献調査から得たデータ
- 先行研究の知見

### 出力
- 構造化された研究計画書
- 論理的に構成された論文・レポート・報告書
- 説得力のあるプレゼンテーション資料

## 注意点

- **「基礎」と「初歩」は異なる**: 本フレームワークは文法（基礎）であり、挨拶の仕方（初歩）ではない。最初は難しく感じるかもしれないが、理解すれば応用がきく
- **型式は歴史的産物**: 現在の論文型式は絶対不変のものではなく、1960〜70年代に広まった歴史的に構築されたもの。理解したうえで使いこなす
- **学問体系の相違を尊重する**: 異なる学問体系には異なる「前提」がある。自分の体系を絶対視せず、比較的な視点を持つ
- **一次資料を重視する**: 二次資料だけでなく、できるだけ自分で原典にあたって確認する
- **完全な論文は存在しない**: 論文を書くことは「不完全さに気づくこと」であり、それ自体が知的成長のプロセスである
