# Skill: 制度-社会乖離分析フレームワーク

## 概要

社会の前提条件が変化したにもかかわらず、旧来の前提に基づいて設計された制度がそのまま運用され続けることで生じる「制度と社会の乖離」を体系的に分析するフレームワーク。吉原祥子『人口減少時代の土地問題』の分析構造から抽出した。

## 適用領域

- 人口動態の変化に伴う制度の機能不全の分析
- 高度成長期に設計された社会インフラ制度の見直し
- 「右肩上がり」前提の法制度が縮小社会で引き起こす問題の構造把握
- 任意制度と義務制度の実効性比較

## フレームワーク構成

### Step 1: 制度の設計前提を特定する

その制度が構築された時代の社会前提を明確にする。

| 分析項目 | 問い |
|----------|------|
| 設計時期 | いつ、どのような社会状況下で制度が設計されたか |
| 前提条件 | 人口増加、経済成長、地価上昇など、何を前提としていたか |
| 設計思想 | 「過剰利用」への対応か、「過少利用」への対応か |
| 義務/任意 | 制度への参加・登録は義務か任意か |

### Step 2: 社会変化との乖離を可視化する

制度の設計前提と現在の社会状況との乖離を定量的に示す。

| 分析項目 | 問い |
|----------|------|
| 前提の崩壊 | 設計時の前提条件は現在も成立しているか |
| 定量的規模 | 乖離はどの程度の規模か（面積、人数、金額など） |
| 進行速度 | 問題は加速しているか、一定か |
| 将来予測 | 現行制度のまま推移した場合、将来どうなるか |

### Step 3: 情報基盤の断片化を検証する

制度が前提とする情報インフラの整備状況を検証する。

| 分析項目 | 問い |
|----------|------|
| 情報の分散 | 関連情報は何種類の台帳/システムに分散しているか |
| 所管の分散 | 所管省庁・部局はいくつにまたがるか |
| 連携の欠如 | 台帳/システム間の情報連携は実現しているか |
| 現場の把握度 | 実務担当者は実態をどの程度把握しているか |

### Step 4: 国際比較で制度の特異性を抽出する

類似の機能を持つ他国の制度と比較し、日本の制度の特異性を明らかにする。

| 分析項目 | 問い |
|----------|------|
| 制度の原理 | 他国はどのような原理に基づいて制度を構築しているか |
| 担い手の違い | 制度の運用を誰が担っているか（行政、専門家、民間） |
| 義務の程度 | 同等の行為は他国では義務化されているか |
| 実効性の仕組み | 義務の実効性を担保する仕組みは何か |

### Step 5: 解決策の3層構造で整理する

問題解決の方向性を「予防」「対処」「基盤」の3層で整理する。

| 層 | 問い | 例（土地問題の場合） |
|----|------|----------------------|
| 予防 | 問題の新規発生をどう防ぐか | 相続登記の義務化・促進 |
| 対処 | 既に発生した問題にどう対応するか | 所有者不明土地の利活用の仕組み |
| 基盤 | 制度を支える情報基盤をどう整備するか | 台帳間連携、地籍調査の推進 |

## 活用例

このフレームワークは以下のようなテーマにも適用できる:

- **社会保障制度**: 現役世代減少に伴う年金・医療制度の持続可能性分析
- **都市インフラ**: 高度成長期に建設されたインフラの老朽化と維持管理問題
- **教育制度**: 少子化時代における学校統廃合と教育アクセスの問題
- **地方自治**: 人口減少下の自治体行政サービスの維持可能性
