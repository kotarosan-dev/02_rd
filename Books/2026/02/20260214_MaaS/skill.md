# Skill: MaaSインパクト評価フレームワーク

## 概要
MaaS（Mobility as a Service）が特定の産業・事業・地域に与える影響を体系的に評価するためのフレームワーク。本書『MaaS モビリティ革命の先にある全産業のゲームチェンジ』から抽出した分析構造をベースとする。

## フレームワーク構造

### 1. MaaSレベル判定（現状分析）

MaaSの成熟度を5段階で評価する。

| レベル | 名称 | 内容 |
|--------|------|------|
| 0 | 統合なし | 各交通事業者が個別にサービス提供 |
| 1 | 情報統合 | マルチモーダルな経路検索が可能 |
| 2 | 予約・決済統合 | 検索・予約・決済がワンストップで完結 |
| 3 | サービス統合 | サブスクリプション型パッケージの提供 |
| 4 | 政策統合 | 都市政策と一体化したMaaSの社会実装 |

**使い方**: 対象地域・サービスが現在どのレベルにあるかを判定し、次のレベルに進むための課題を特定する。

### 2. ステークホルダー・マッピング

MaaSに関わるプレーヤーを6つのカテゴリで整理する。

1. **MaaSオペレーター**: サービス統合を担う中間プレーヤー（例: MaaS Global、MONET Technologies）
2. **交通サービスプロバイダー**: 鉄道、バス、タクシー、カーシェア、自転車シェア等の事業者
3. **テクノロジープロバイダー**: API、決済、データ分析等の技術基盤を提供する企業
4. **行政・規制当局**: 交通政策の策定、規制緩和、データ公開義務化を担う
5. **Beyond MaaSプレーヤー**: 不動産、小売り、医療、観光等の異業種
6. **エンドユーザー**: 移動サービスの利用者（都市住民、観光客、高齢者等）

**使い方**: 各カテゴリのプレーヤーをリストアップし、それぞれの利害・動機・障壁を分析する。

### 3. 三層変化モデル（MaaS変革の波及構造）

MaaSがもたらす変化を三層で捉える。

| 層 | 変化の内容 | 影響範囲 |
|----|-----------|---------|
| 第一層: 移動のパーソナライズ化 | 個々人のニーズに合わせた移動手段のアレンジ、新たな移動需要の創出 | 交通事業者、テクノロジー企業 |
| 第二層: 都市交通の全体最適化 | モビリティの統合制御、定額パッケージによる他産業とのワンパッケージ化 | 都市計画、行政、エネルギー |
| 第三層: 都市・場所の再定義 | 駐車場消滅による空間の再活用、立地によらないビジネスの可能性 | 不動産、小売り、医療、保育 |

**使い方**: 自社・対象事業がどの層の変化から影響を受けるかを特定し、対応戦略の優先順位を決める。

### 4. 通信革命アナロジー分析

MaaSの構造変化を通信業界の変遷と対比して分析する。

| 通信業界の変遷 | MaaS領域の対応 |
|---------------|--------------|
| 従量課金 → 定額制 | 交通料金の従量制 → 定額乗り放題パッケージ |
| コンテンツの個別販売 → パッケージ化 | 交通サービスの個別提供 → 統合パッケージ |
| GAFA等の巨大プラットフォーマーの出現 | 交通版プラットフォーマーの出現可能性 |
| OTTサービスの急成長 | MaaS上の新たな価値産業の創出 |

**使い方**: 自社の位置を「コンテンツプロバイダー」「プラットフォーマー」「OTTサービス」のいずれかに当てはめ、プラットフォーム化の波にどう対応するかを検討する。

### 5. Beyond MaaS 産業影響度チェックリスト

自社事業へのMaaS影響度を評価する8つの問い。

- [ ] 自社の顧客は移動を伴ってサービスを利用しているか？
- [ ] 自社の立地・拠点配置は「交通の便」に依存しているか？
- [ ] 自社のサービスに移動パッケージを組み込むことで付加価値が生まれるか？
- [ ] 自社の顧客データと移動データを掛け合わせた新サービスが構想できるか？
- [ ] 自動運転・EVの普及が自社のコスト構造を変える可能性があるか？
- [ ] MaaS定額パッケージにより、自社の顧客の来訪頻度が変化し得るか？
- [ ] 駐車場や交通アクセスの制約がなくなった場合、出店・拠点戦略が変わるか？
- [ ] 自社がモビリティサービスを直接提供する可能性があるか？

**使い方**: 4つ以上「はい」がある場合、MaaSへの戦略的対応が必要。6つ以上の場合、事業モデルの根本的な見直しを検討すべき。

## 適用場面

- 新規事業検討時のMaaS関連機会の評価
- 都市交通計画・地域公共交通計画の策定
- 異業種連携によるMaaS関連サービスの企画
- 自動車・交通産業の中長期戦略立案
- 不動産・小売り・医療等における立地戦略の見直し
