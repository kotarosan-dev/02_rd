# 物語の書き出し設計フレームワーク

## 概要

レス・エジャートン著『「書き出し」で釣りあげろ』に基づく、小説・物語の冒頭を効果的に設計するためのフレームワーク。読者を1ページ目から引き込み、離さない書き出しを作成する。

## 使用場面（When to Use）

- 小説・短編小説の冒頭を執筆するとき
- 既存の書き出しを改善・リライトするとき
- 物語のプロット設計段階で冒頭を構想するとき
- 脚本やゲームシナリオの導入部を設計するとき
- 他者の作品を分析・批評するとき

## 書き出しの5つの構成要素

### 1. きっかけとなる出来事（Inciting Incident）

**定義**: 主人公の安定した世界を揺るがし、ストーリー上の問題を発生させる出来事

**要件**:
- 物語の最初のシーンに配置する
- 主人公の人生に重大な変化をもたらす
- 後戻りできない状況を作り出す
- 核心の問題への入り口となる

**例**:
- 木から逆さ吊りにされて目覚める（『アルアル島の大事件』）
- 強盗が店に押し入る（『ヒストリー・オブ・バイオレンス』）

### 2. 表層の問題（Surface Problem）

**定義**: ストーリーの表面で展開される具体的・即時的な課題

**要件**:
- きっかけとなる出来事から直接発生する
- 読者が即座に理解できる具体的な危機
- 主人公が即座に対処を迫られる
- 核心の問題を徐々に露呈させる手段となる

**例**:
- 人食い人種に食べられる危険
- 強盗に対して行動を起こすかどうかの判断

### 3. 核心の問題（Story-Worthy Problem）

**定義**: 主人公の内面的・心理的な課題。物語の真のテーマ

**要件**:
- 書き出しでは暗示にとどめる
- 物語を通じて徐々に明らかになる
- 主人公の人生や価値観に関わる深い問題
- 最終的な解決が物語のクライマックスとなる

**例**:
- 人間的な未熟さと浮ついた生き方（タッカー）
- 裏社会の過去を家族に受け入れてもらうこと（トム）

### 4. 設定（Setup）

**定義**: きっかけとなる出来事を理解するために必要最小限の舞台情報

**要件**:
- 最小限に抑える（数行〜数段落）
- シーンを理解するのに必要な情報のみ
- バックストーリーとは区別する
- きっかけとなる出来事と同時に提示可能

### 5. バックストーリー

**定義**: きっかけとなる出来事以前に起こった過去の出来事

**配置ルール**:
- きっかけとなる出来事の**後**に配置
- 読者が興味を持ってから提供
- フラッシュバックまたは自然な形で織り込む
- 必要最小限に抑える

## 避けるべき書き出し（注意信号）

| 注意信号 | 問題点 | 代替案 |
|----------|--------|--------|
| 夢で始める | 読者の感情的投資が無駄になる | 現実のシーンで始める |
| 目覚まし時計が鳴る | きっかけとなる出来事がない | 危機的状況から始める |
| 長い描写・説明 | 読者を待たせる | アクションで見せる |
| 会話で始める | 登場人物が確立されていない | 人物を先に確立する |
| とりとめのない思考 | 読者が感情移入できない | ドラマチックなシーンで見せる |

## 書き出し設計プロセス

### Step 1: 核心の問題を決める
物語全体のテーマとなる主人公の内面的課題を明確にする。

### Step 2: 表層の問題を設計する
核心の問題を露呈させる具体的・外面的な課題を設計する。

### Step 3: きっかけとなる出来事を創造する
表層の問題を発生させ、主人公を行動へと駆り立てる出来事を創造する。

### Step 4: 最小限の設定を決める
シーンを理解するのに必要な情報のみを特定する。

### Step 5: バックストーリーの配置を計画する
どの情報を、いつ、どのように提供するかを計画する。

### Step 6: 「遅れて来て、早く離れる」を適用する
シーンの途中から物語を始め、必要に応じてバックストーリーに移行する。

## 作品分析テンプレート

作品を分析する際は以下の項目を特定する：

```
作品名：
著者：

【きっかけとなる出来事】


【表層の問題】


【表層の問題の帰着点】


【核心の問題】


【核心の問題の帰着点】


【設定の提示方法】


【バックストーリーの配置】

```

## 自己チェックリスト

書き出しが完成したら確認：

- [ ] きっかけとなる出来事が最初のシーンにあるか
- [ ] 読者が理解できる表層の問題があるか
- [ ] 核心の問題が暗示されているか
- [ ] バックストーリーは最小限か
- [ ] 主人公の人物像が確立されているか
- [ ] 読者を待たせていないか
- [ ] 注意信号を含んでいないか
- [ ] 先を読みたいと思わせるか

## 注意点

- 本フレームワークは主に商業小説向け。純文学・実験的文学には別の基準がありうる
- 「ルール」ではなく「指針」として活用する
- 例外的に成功する書き出しも存在する
- 最終的には物語と読者に奉仕することが目的
