# 失敗から学ぶ技術 ー 新規事業開発を成功に導くプロトタイピングの教科書

**著者**: 三冨敬太（Keita Mitomi）

---

## 1. 一言で言うと

**「プロトタイピングとは、仮説と現実のギャップを早期に発見し、小さな失敗を先取りすることで大きな失敗を防ぐ技術である」**

---

## 2. 主要なポイント

### ポイント1: プロトタイピングの定義と分類
- **プロトタイプ** = 完成する前のモノや体験
- **プロトタイピング** = プロトタイプを活用したプロセス・手法
- 分類は「価値」「実現可能性」「ルック&フィール」「インテグレーション」の4つ
- **初期段階ではまず「価値」のプロトタイピングから始める**ことが重要

### ポイント2: プロトタイピングの3つの役割
1. **コミュニケーション**: チーム内外の認識を共通化し、円滑な協働を促進
2. **学習**: つくる・体験する過程で新しい気づきを獲得
3. **意志決定**: 進むか止まるかの判断材料を提供

### ポイント3: 行動原則「早く・安く・何度も・並行して」
- **早く試す**: 早期にプロトタイピングしたチームほど良い結果を残す
- **安く試す**: 破棄前提の安価なプロトタイプでも80%の問題を検出可能
- **何度も試す**: 反復することで気づきが増え、成績が向上する
- **並行して試す**: 複数のアイデアを同時に試すことで視野が広がる

### ポイント4: プロトタイピングのプロセス「設計→構築→評価」
- **設計**: ゴール・スケジュール・仮説・リソース・テスト方法を決定
- **構築**: チラシ、ダーティーエクスペリエンス、ワイヤーフレームなど10種類の手法
- **評価**: インタビュー・観察・アンケートでデータ収集し、仮説を検証

### ポイント5: 認知バイアスへの対策
- サンクコストバイアス（投資したからもったいない）や確証バイアス（都合の良い情報だけ集める）がアイデアの修正を妨げる
- **早くプロトタイピングすることで、これらのバイアスが強くなる前に軌道修正できる**

## 3. 実践への示唆（明日から使える3つのアクション）

### アクション1: 「とりあえずやってみる」を合言葉にする
- プロトタイピング・マインドセットは「やりながら獲得できる」
- 完璧を求めず、30分でチラシを1枚つくってみることから始める
- やらない理由（上司への説明、恥ずかしさ）を無視して手を動かす

### アクション2: プロトタイピングシートを活用する
以下の5項目を15〜30分で記入し、プロトタイピングを設計する:
1. 確認したい仮説
2. つくるモノ・体験すること（What）
3. 構築方法（How）
4. 仮説の評価方法（データ収集・評価基準）
5. 評価結果・気づき

### アクション3: ユーザーインタビューを早期に実施する
- 価値を感じるユーザーが自分でない場合、当事者への確認は必須
- 知り合い・SNS・クラウドソーシングなど複数の募集方法を並行活用
- オンラインインタビューで移動時間を削減し、スピードを優先

## 補足: 10種類のプロトタイプ（価値のプロトタイピング向け）

| 種類 | 用途・特徴 |
|-----|-----------|
| A. チラシ | 30分で作成可能。ユーザーに見せてフィードバックを得る |
| B. ダーティーエクスペリエンス | 既存のモノを組み合わせて粗く体験する。新規性が高いアイデアに有効 |
| C. 工作 | 段ボール・紙などでフィジカルにつくる |
| D. スケッチ | 紙とペンで最速でアイデアを可視化 |
| E. ムードボード | 写真・イラストを組み合わせてイメージを共有 |
| F. LP（ランディングページ） | URLを発行し、多くのユーザーからフィードバックを得る |
| G. 動画 | コンセプトをわかりやすく伝える。クラウドファンディング向け |
| H. ペーパーワイヤーフレーム | 紙で簡易的に画面イメージを作成 |
| I. 画面遷移ワイヤーフレーム | Figma等で画面遷移を設定。デジタルアイデアに必須 |
| J. システム連携ノーコード | Bubble等で実際に近い動作を実現。精度の高い検証向け |
