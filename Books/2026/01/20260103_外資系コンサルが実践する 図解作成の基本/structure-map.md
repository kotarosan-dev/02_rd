## 判断マップ

### 1. まず何を直すべきか

| 症状 | 最初に見る場所 | 打ち手 |
|------|----------------|--------|
| ごちゃごちゃして読みにくい | Level 1 | 枠線・影・色数・ズレを削る |
| どこが重要か分からない | Level 2 | 強調を1か所に絞る |
| 図にしたのにしっくりこない | Level 3 | 情報構造と型のズレを疑う |
| なんとなくダサい | 基本原則 + Level 1 | ノイズと整列の不足を疑う |

### 2. 情報構造から型を選ぶ

| 情報の種類 | 向く型 |
|------------|--------|
| 一覧整理 | マトリクス図 |
| 優先順位・評価 | 象限図 |
| 階層・分解 | ロジックツリー / ピラミッド |
| 因果関係 | フィッシュボーン |
| 手順・工程 | プロセス図 |
| 循環 | サイクル図 |
| 比較 | 棒グラフ |
| 割合 | 円グラフ |
| 時系列 | 折れ線グラフ |
| 相関 | 散布図 |

## 圧縮方針メモ

- `source.txt` 自体が高圧縮だったため、章本文の再現より「フレームワークの再配置」を優先した。
- 原著の図版差分までは保持できないので、圧縮では完了条件・判断基準・失敗パターンを強く残した。
- ジャンル判定は、図解テーマでG要素があるものの、実際の核はプレゼン実務の原則提示なのでCを主判定とした。
