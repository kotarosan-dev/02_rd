## コアフレームワーク {#core}

### グラフ理論：問題ユニバースの地図

```
グラフ G = (V, E)
│
├─ ルート最適化問題
│   ├─ 最短経路 ─── Dijkstra O(n²)  ← 単一始点、非負重み
│   │               Bellman-Ford   ← 負重み可、O(nm)
│   │               Warshall-Floyd ← 全対、O(n³)
│   └─ NP困難な経路
│       └─ 「最大でも一度だけ通過を許す最短経路」→ NP困難
│
├─ 全域木問題（Spanning Tree）
│   ├─ Prim法    O(n²) ── Dijkstraと兄弟、MST
│   └─ Kruskal法 O(m log m) ── 辺をソートして貪欲
│
├─ オイラー問題（辺を全部通る）
│   ├─ オイラー閉路 ←→ 全ノードの次数が偶数
│   ├─ オイラー小路 ←→ 次数が奇数のノードがちょうど2個
│   └─ 中国の郵便配達員問題 → 奇数次数ノードを最小コストでペア → O(m²)
│
├─ ハミルトン問題（ノードを全部通る）
│   ├─ ハミルトン路・閉路の存在判定 → NP困難（十分条件定理なし！）
│   └─ TSP（最短ハミルトン閉路を求める最適化版） → NP困難
│       ├─ 構成的ヒューリスティック（最隣接法）
│       ├─ 改良的ヒューリスティック（ノード挿入法）
│       ├─ クリストファイズ法：最適解の≤1.5倍保証
│       └─ 分枝限定法：上限（ヒューリスティック）＋下限（1-木）で枝刈り
│
└─ 計算複雑度理論
    ├─ P問題（多項式時間で解ける）
    ├─ NP問題（解を多項式時間で検証できる）
    ├─ NP困難（NP問題中の最も難しいクラス）
    └─ P=NP問題（未解決、クレイ数学賞100万ドル）
```

### 本書の数学的骨格：三大定理

| 定理 | 内容 | 証明手法 |
|---|---|---|
| **オイラーの定理** | グラフがオイラー閉路を持つ ⟺ 全ノードの次数が偶数 | 次数の性質＋構成的証明 |
| **次数の偶数性** | 任意のグラフで奇数次数ノードの個数は常に偶数 | **完全帰納法**（数学的帰納法）で証明 |
| **クリストファイズ定理** | 三角不等式を満たすTSPに対し、最適解の≤1.5倍のツアーを保証 | 1-木＋最小マッチング＋オイラー路 |

## 章別エッセンス {#chapters}

### 第1章：ハンブルクへの旅（Das 1. Abenteuer）
**数学的内容**：グラフの動機付け。ハンブルクまでの最短ドライブルートを求める問題から、「経路」の数学的モデルとしてのグラフが登場。バイエルン→チェコ→ハンブルクなどの地図をグラフ化。

**物語**：パパからAI搭載パソコンを贈られた15歳のレナ。AI「ビム（Bim）」との会話が始まる。

> 「数学なんて何の役に立つの？」——レナの最初の言葉が、この本全体のテーゼ。

---

### 第2章：グラフ理論の基礎（Graphen）
**数学的内容**：
- **ノード（Node）**：頂点。例：交差点、都市
- **辺（Edge）**：2ノード間の接続。例：道路
- **重み付きグラフ**：辺に数値（距離・コスト）を付加
- **有向グラフ（Digraph）**：辺に向きがある（一方通行）
- **多重グラフ**：2ノード間に複数の辺が存在

**核心**：現実問題の数学的モデル化＝グラフ化。ミュンヘンのSバーンの路線図がグラフの具体例として登場。

---

### 第3章：最短経路問題の難しさ（Kombinatorische Explosion）
**数学的内容**：組合せの爆発。n都市の全経路数は n! 。n=20なら 2,432,902,008,176,640,000 通り。「力技（ブルートフォース）」は現実的でない。

**核心思考**：探索空間が爆発的に増える問題には「賢いアルゴリズム」が必要。

---

### 第4章：ダイクストラのアルゴリズム（Dijkstras Algorithmus）
**数学的内容**：ダイクストラ法（1959）の手順。

```
初期化：全ノードの暫定距離 = ∞、始点 = 0
繰り返し：
  1. 未確定ノードの中で最小暫定距離のノード u を選ぶ
  2. u の各隣接ノード v について：
     もし d[u] + w(u,v) < d[v] なら d[v] を更新
  3. u を「確定済み」にする
終了条件：全ノード確定、または目標ノード確定
```

**注意点**：**負の辺の重みがあると失敗**。なぜなら「一度確定したノード」に後から短い経路が現れる可能性があるから。

---

### 第5章：ダイクストラの正当性（Korrektheit）
**数学的内容**：なぜダイクストラが正しいか。「確定済みノードには最短経路が確定している」という不変条件（ループ不変式）の維持を直感的に示す。

**重要概念**：アルゴリズムの「証明」と「直観的理解」の区別。

---

### 第6章：計算複雑度・Big O記法（Ordnung muss sein）
**数学的内容**：
- **O(n)記法**：入力サイズnに対する最悪計算時間の上界
- **フォルダ分類**（本書独自の比喩）：
  - O(1)フォルダ：定数時間
  - O(log n)フォルダ：対数時間
  - O(n)フォルダ：線形時間
  - O(n²)フォルダ：ダイクストラが属する
  - O(2ⁿ)フォルダ：組合せの爆発（指数時間）

**ビムの言葉**：「O(n²)フォルダなら、nが10倍になっても計算時間は100倍。でもO(2ⁿ)フォルダなら…永遠に終わらない」

### 第7章：アラン・チューリングとチューリングマシン（Alan Turing）
**数学的内容**：
- チューリングマシン：無限テープ上を読み書きする仮想的な計算機
- すべての現代コンピューターはチューリングマシンと等価
- チューリングはナチスのエニグマ暗号解読に貢献し、コンピューター科学の父と呼ばれる

**物語**：映画「2001年宇宙の旅」のHAL 9000がビムに似ている、というレナの感想。

---

### 第8章：前処理と計算効率化（Vorverarbeitung）
**数学的内容**：同じ問題を繰り返し解く場合、事前にデータを整理（前処理）しておくことで全体の計算量を削減できる。例：ミュンヘン交通網で毎日最短経路を計算するなら、前処理が大幅な高速化をもたらす。

**現実応用**：カーナビ、Google Maps等の背景技術。

---

### 第9章：木と全域木（Bäume）
**数学的内容**：
- **木（Tree）**：連結していて閉路がないグラフ
- **全域木（Spanning Tree）**：グラフGのすべてのノードを含む木
- **n個のノードの木は必ずn-1本の辺を持つ**（重要性質）
- **最小全域木（MST）**：辺の重みの合計が最小の全域木

**応用**：電話網、電力網、コンピューターネットワークの最小コスト接続設計。

---

### 第10章：プリム法（Prims Algorithmus）
**数学的内容**：MST構築アルゴリズム。

```
初期化：任意のノードから始め、「木」T を作る
繰り返し：
  T に含まれるノードから、T に含まれないノードへの辺のうち
  最小重みの辺を T に追加
終了：全ノードが T に含まれる
```

**ダイクストラとの比較**：同じ「欲張り型」構造。違いは「最短距離」ではなく「木への最安値接続」を基準とすること。

---

### 第11章：クルスカル法（Kruskals Algorithmus）
**数学的内容**：別のMSTアルゴリズム。辺をすべて重み順にソートし、閉路を作らない辺を順に加える。

**貪欲アルゴリズムとマトロイド**：なぜ「欲張り」が最適解を保証するか。その数学的根拠はマトロイド理論。全域木問題はマトロイドの公理を満たすため、貪欲法が機能する。

---

### 第12章：有向グラフと最小有向全域木（Arboreszenzen）
**数学的内容**：
- **有向全域木（Arborescence）**：根から全ノードへちょうど一本の有向経路が存在する有向グラフ
- 無向グラフのMSTに対応する有向版はより複雑（Edmonds法）

---

### 第13章：負の辺とワーシャル・フロイド（Warshall-Floyd）
**数学的内容**：
- 負の辺の重みがある場合のダイクストラの失敗例
- **ワーシャル・フロイド法**：O(n³)で全ノード対間の最短経路を求める動的計画法

```
初期化：d[i][j] = 辺(i,j)の重み（なければ∞）
for k = 1 to n:
  for i = 1 to n:
    for j = 1 to n:
      d[i][j] = min(d[i][j], d[i][k] + d[k][j])
```

- **負の閉路検出**：d[i][i] < 0 なら負の閉路が存在

### 第14章：モデル化の実際（Modellierung）
**数学的内容**：現実問題のグラフ化における設計判断。「ローテンブルクへの立ち寄り」問題を題材に、寄り道を「負の辺の重み」でモデル化するアイデア。

**伏線**：この「負の重み・一度だけ通過」問題が後のNP困難性の議論で重要になる。

### 第15章：最短経路問題の変種（Varianten）
**数学的内容**：
- 開始・終了が同じノードの最短閉路問題
- 各ノードを「最大でも一度だけ」通過する制約付き最短経路
- これらはすべてダイクストラでは解けず、後にNP困難と判明する

---

### 第16章：数学的帰納法（Vollständige Induktion）
**数学的内容**：証明手法の解説。

**証明の構造**：
1. **帰納法の出発点** A(0)：m=0（辺なし）のとき命題が成立
2. **帰納法の仮定**：A(m)が成立すると仮定
3. **帰納法のステップ**：A(m) → A(m+1)を示す

**具体的証明**：「任意のグラフで、次数が奇数のノードの個数は常に偶数」を帰納法で証明。辺を1本加えるとき、両端のノードの次数がそれぞれ+1され、奇数ノードの数の偶奇が維持されることを示す。

---

### 第17章：ケーニヒスベルクの橋（Königsberger Brücken）
**数学的内容**：歴史的問題（1736年、オイラー解決）。プレーゲル川の7つの橋を、それぞれ一度だけ渡ってすべて通行できるか？

**グラフ化**：島・岸を4ノード、橋を7辺とするグラフ。4ノードすべての次数が奇数（3,3,3,5）→ **オイラー小路も存在しない**。

**オイラーの定理**：
- 連結グラフGにオイラー閉路が存在 ⟺ 全ノードの次数が偶数
- 連結グラフGにオイラー小路が存在 ⟺ 次数が奇数のノードがちょうど2個

---

### 第18章：完全帰納法でオイラー定理を証明（Beweis durch vollständige Induktion）
**数学的内容**：第16章の帰納法を実際に使い、「奇数次数ノードは常に偶数個」を厳密に証明。三つのケース分析（辺を追加するとき両端が奇奇/偶偶/奇偶の場合）で、辺の追加に関わらず奇数次数ノードの個数の偶奇が保存されることを示す。

---

### 第19章：オイラー路の構成アルゴリズム（Eulersch oder nicht）
**数学的内容**：オイラー路の存在判定（次数チェック）と構成アルゴリズム。

1. 開始ノードから進めなくなるまで進む
2. 未使用の辺を持つノードを選び、新しい閉路を構成
3. 既存の経路に組み込む

**サンタクロースの家**：ドイツの伝統的な一筆書きパズル。2つの下端ノード（次数3）から始めれば完成できる——オイラー定理の直接応用。

### 第23章：中国の郵便配達員問題（Post aus China）
**数学的内容**：管梅谷（1962）によるアルゴリズム。

```
中国の郵便配達員問題アルゴリズム
Input: 重み付きグラフ G = (V,E)
Output: 全辺を含む最小閉路

1. 次数が奇数のノードの集合 V' を求める
2. V' の任意の2ノード間の最短経路の長さを求める（ダイクストラ活用）
3. V' のノードで完全グラフを構成し、最適マッチング M を求める
4. M の辺を G に追加して多重グラフ G'' を構成する
5. G'' のオイラー閉路を求める
```

**応用**：ゴミ収集、郵便配達、除雪車、プロッタの描画経路最適化。

### 第24章：チェックメイト——ナイト跳び（Schach-Matt?）
**数学的内容**：

### 第25章：プラトニックな愛?——正多面体（Platonische Liebe?）
**数学的内容**：
- **ハミルトン路問題**：すべてのノードを一度だけ通る経路
- オイラー路（辺を全部通る）との根本的な違い：**ハミルトン路には「オイラーの定理」に相当する十分条件の定理がない**

- ハミルトンのゲーム「二十ゲーム（icosian game）」：正十二面体のハミルトン閉路を見つけるゲーム

### 第26章：NP困難——表記上の問題（Notorisch Problematisch）
**数学的内容**：計算複雑度理論の核心。

| 概念 | 定義 |
|---|---|
| **決定問題** | 答えが「はい/いいえ」の問題 |
| **NP問題** | 「はい」という答えを多項式時間で**検証**できる問題 |
| **NP困難** | すべてのNP問題を多項式時間で帰着できる問題 |
| **P=NP問題** | 「多項式時間で解ける問題」とNP問題が一致するか |

**帰着の論理**：問題Aが問題Bに帰着できる（B→A）なら、AのアルゴリズムでBも解ける。一つのNP困難問題を解く効率的アルゴリズムがあれば、全NP問題が効率的に解ける。

**充足可能性問題（SAT）**：スティーブン・クック（1971）が最初のNP困難問題として示した。

---

### 第27章：巡回セールスマン問題（巡回セールスマンのためだけではなくて）
**数学的内容**：
- **巡回セールスマン問題（TSP）**：すべてのノードを一度だけ通る最短閉路
- NP困難の代表例
- **最隣接ヒューリスティック（Nearest Neighbor）**：次に近いノードへ進む貪欲法
  - 欠点：局所最適にはまり込む。誤差が任意に大きくなりうる（1兆倍になることも理論上可能）
- **応用**：保険外交員のルート最適化、半導体基板のドリル穿孔順序（20%の生産性向上！）

---

### 第28章：上限と下限（少ないは多い）
**数学的内容**：
- **上限（Upper Bound）**：ヒューリスティック解法で見つけたツアーの長さ（最適解より長い可能性）
- **下限（Lower Bound）**：緩和法（relaxation）で得る。問題の制約を「緩める」ことで素早く計算
- **1-木（1-Tree）**：全域木からノードを一つ外した残りの最小全域木に、外したノードを2本の最短辺で再接続した構造
  - 各TSPツアーは1-木であるが、各1-木はTSPツアーではない
  - 最小1-木の長さ ≤ 最適TSPツアーの長さ → **下限**として使える

---

### 第29章：クリストファイズのアルゴリズム（150-prozentig）
**数学的内容**：品質保証付きの近似アルゴリズム（ニコス・クリストファイズ）。

```
クリストファイズのアルゴリズム
Input: 三角不等式を満たす重み付き完全グラフ G
Output: 最適TSPツアーの≤1.5倍のツアー

1. G の最小1-木 B を求める
2. B で次数が奇数のノードの最適マッチング M を求める
3. B と M の辺を通るオイラー閉路を求める
4. オイラー閉路で複数回通過するノードを、直接接続辺で「ショートカット」する
```

**なぜ1.5倍が保証されるか**：
- 1-木の辺の合計 ≤ 最適ツアーの長さ
- 最小マッチングの辺の合計 ≤ 最適ツアーの半分（2つのマッチングが最適ツアーを分割するから）
- 合計≤1.5倍

---

### 第30章：分枝限定法（ボンサイ）
**数学的内容**：NP困難問題を解くための基本的枠組み。

```
分枝限定法（Branch & Bound）
1. ヒューリスティックで初期上限を得る
2. 現在のノードの下限を計算（最小1-木など）
3. 下限 ≥ 現在の最良上限 → 枝刈り（STOP）
4. そうでなければ枝分岐して子ノードを展開
5. 完全ツアーが見つかれば上限を更新
```

**ポイント**：組合せ爆発を「刈る」ことで、小〜中規模問題では最適解が現実的時間で得られる。

---

### 第31章：多面体組合せ論（全然、プラトニックでない）
**数学的内容**：
- TSPのツアーを0/1ベクトルで表現し、n辺のグラフなら n次元空間の点として扱う
- 全ツアーの集合が形成する多面体（**TSP多面体**）の頂点が最適ツアー
- **多面体を「面（不等式）」で表現**することで、頂点を直接列挙せずに問題を解ける
- n=100のTSP多面体は31桁の頂点数だが、面数は200個程度（立方体の場合）
- 分枝限定法の下限として「チーズを切り出す」（有効不等式を生成）

**ダンツィク（1947）**：線形計画法と多面体の最適頂点を見つける高速手法（シンプレックス法）を開発。

### 引用1：数学の意外な普遍性

## 批判的視点 {#critical}

### 本書の強み
1. **数学的内容の正確性**：ダイクストラ、クルスカル、クリストファイズ等のアルゴリズムを正確に記述。1.5倍近似の証明も本質的に正しい。
2. **物語の一貫性**：「なぜこの概念が必要か」の動機付けが各章で丁寧に行われる。
3. **現代の応用との接続**：半導体ドリル穿孔、ゴミ収集、プロッタ等の実務応用が豊富。

### 本書の弱みと注意点
1. **図表なしでは追いにくい数式**：圧縮読本では数式・図が省略されているため、特に「次数が奇数のノードのペアリング」や「1-木の証明」は原書の図を見ないと腑に落ちにくい。

4. **ハミルトン問題の深みが浅い**：「なぜ十分条件がないのか」の本質（位相幾何学的理由）には踏み込まない。

## 読む上でのピットフォール {#pitfalls}

### ピットフォール1：「ダイクストラ法は常に使える」
→ **誤り**。負の辺の重みがあると誤った最短経路を返す可能性がある。負辺が存在する場合はベルマン・フォード法かワーシャル・フロイド法を使う。

### ピットフォール2：「最小全域木とダイクストラ最短経路ツリーは同じ」
→ **誤り**。MST（全域木の辺の重み合計最小化）と最短経路ツリー（始点から各ノードへの距離最小化）は異なる目的関数を持つ。プリム法とダイクストラ法は構造が似ているが答えが違う。

### ピットフォール3：「次数が奇数のノードが偶数個 → オイラー閉路が存在」
→ **不完全**。正確には「グラフが連結していて、かつ全ノードの次数が偶数 ⟺ オイラー閉路が存在」。連結性の条件が欠かせない。

### ピットフォール4：「ハミルトン路もオイラー定理のような十分条件定理がある」
→ **誤り**。これが本書最大の驚きポイント。ハミルトン路の存在を効率的に判定する一般的な定理は存在しない（NP困難）。ディラックの定理など十分条件はあるが、充分条件にすぎない。

### ピットフォール5：「NP困難な問題には有用なアルゴリズムがない」
→ **誤り**。「効率的な厳密解法」がないだけで、ヒューリスティック法・近似アルゴリズム・分枝限定法など実務で使える手法が豊富に存在する。TSP問題は実際に数万都市規模で最適解が得られている。

### ピットフォール6：「クリストファイズ法は1.5倍の誤差がある」
→ **精密化が必要**。クリストファイズ法は「最悪でも1.5倍」の保証をするが、実際にはほとんどの場合もっとよい結果が出る。本書中の小例でも15%程度の誤差。

### ピットフォール7：「TSP世界記録は本書記載のもの」
→ **情報が古い**。本書は2000年代初頭の記述。現在はさらに大規模なTSPが解かれており、アトランタのGeorgia Tech TSPチームが継続的に更新している（2006年以降）。
