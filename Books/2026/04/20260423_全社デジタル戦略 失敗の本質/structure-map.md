# 構造マップ：全社デジタル戦略 失敗の本質

> BCGコンサルタントが診断する「日本企業IT投資失敗の構造」と処方箋

---

```
問題提起（はじめに）
「日本企業は欧米比3倍の工数・DX成功率1.4%」
         ↓
現状診断（第1章）
「なぜ失敗するのか：業界別実態 → 2025年の崖 → 人材問題」
         ↓
前提整備（第2章）
「準備：4つのマインド → CIO/CDO/CHRO → 内製化戦略」
         ↓
実行方法論（第3章）
「3フェーズ（企画構想・要件定義・実装）の落とし穴と対策」
         ↓
学習と回復（第4章）
「失敗の3分類 → 軌道修正の判断基準 → 失敗ナレッジの制度化」
         ↓
勝ち筋（第5章）
「スタンダードデータ・業界連携・生成AI・人材育成・国の役割」
         ↓
結論（おわりに）
「技術は変わるが変革を主導する人間の役割は増す」
```

## 章別構造マップ

### はじめに
- **目的**：読者への問題意識の共有
- **主張**：失敗の本質は技術ではなく、意思決定と組織文化にある
- **引用元**：「失敗の本質 日本軍の組織論的研究」との接続
- **読者対象**：経営者・IT部門長・DX推進ビジネスパーソン

---

### 第1章：なぜ日本企業はIT投資で失敗するのか

```
┌─────────────────────────────────────────────────┐
│ 業界別IT投資状況                                  │
│   小売：ファーストリテイリング（RFID・AI需要予測） │
│   飲食：すかいらーく・スターバックス              │
│   金融：地方銀行レガシー・大手保険100億円中止     │
│   製造：「翻訳力」不足の問題                      │
├─────────────────────────────────────────────────┤
│ 構造的問題                                        │
│   2025年の崖（年間最大12兆円損失）                │
│   IT人材の偏在（74%がITベンダー所属）             │
│   ERPカスタマイズ率95%超                          │
│   ERP導入期間：日本3〜5年 vs 海外1〜2年           │
├─────────────────────────────────────────────────┤
│ 根本原因                                          │
│   「オペレーションドリブン」という思想の問題       │
│   → 現行業務優先 → カスタマイズ過多 → 負のスパイラル│
└─────────────────────────────────────────────────┘
```

**キー数字**：
- DX成功率：日本1.4%（世界平均30%）
- IT人材：日本約140万人・70%がITベンダー所属
- CIO設置率：15.9%、CDO：16.6%

---

### 第2章：IT・デジタルプロジェクト成功に向けての準備

```
準備の3要素
    ├── デジタルリテラシー（経営者が持つべき「5つの問い」）
    ├── 4つのマインド
    │       ① 戦略連動
    │       ② アウトカム志向
    │       ③ アジャイル思考
    │       ④ データ資産化
    └── 組織・体制
            ├── CIO（設置率15.9%）
            ├── CDO（設置率16.6%）
            ├── CHRO（CIO/CDOと連携）
            └── 内製化（日本20%超、1000人超で40%）
```

---

### 第3章：プロジェクトの各フェーズでやるべきこと

```
フェーズ1：企画・構想
  主な罠: 目的の曖昧化、スコープクリープ、ROI楽観設定
  対策: AS-IS/TO-BEマッピング、価値仮説の文書化

フェーズ2：要件定義
  主な罠: 現行業務の写し、ユーザー不在、「あとで調整」の先送り
  対策: 業務部門キーパーソンの正式参加、RFP質の向上

フェーズ3：実装・導入
  主な罠: スコープクリープ再発、テスト工程軽視、移行リスク軽視
  対策: ゲートレビュー制度、変更管理の前倒し
```

---

### 第4章：失敗から学び失敗を活用する

```
失敗の3分類
    ├── 回復可能な失敗 → 迅速な判断・ピボット
    ├── 高コストな失敗 → 徹底的な原因分析と共有
    └── 壊滅的な失敗  → 予防設計・リスク管理強化

軌道修正の判断基準（数値的閾値）
    ├── コスト：当初見積もりの150%超 → 強制全体見直し
    ├── スケジュール：6ヶ月以上遅延 → ROI仮説再検証
    └── ビジネス環境：大きな変化 → 計画前提を再点検

失敗ナレッジの制度化
    ├── Post-mortemの義務化
    ├── 失敗事例データベース構築
    └── 心理的安全性の醸成
```

---

### 第5章：日本企業が勝つためのアプローチ

```
勝ち筋の4軸
    ├── スタンダードデータへの転換
    │       MDM・データモデル標準化・API連携
    ├── 業界連携・コンソーシアム戦略
    │       ITインフラで協調・顧客体験で差別化
    ├── 生成AI活用
    │       大手保険：開発工数50%低減・欠陥30%低減
    │       （データ品質・ドメイン知識が前提）
    └── IT人材育成
            デジタルゼネラリスト
            DXプロデューサー
            データサイエンティスト
```

## 関係性マップ：失敗の構造

```
【根本原因】
オペレーションドリブン思想
    │
    ├→ ERPカスタマイズ過多（日本95%超）
    │       ↓
    │   維持コスト膨張・新機能追加困難
    │
    ├→ IT人材の偏在（74%がベンダー）
    │       ↓
    │   要件定義能力の欠如
    │   「ベンダー任せ」固定化
    │
    ├→ CIO/CDO不在（設置率15〜16%）
    │       ↓
    │   IT戦略の全社最適化ができない
    │   ガバナンス欠如
    │
    └→ 「空気」による意思決定
            ↓
        埋没コストバイアス
        失敗プロジェクトの継続
        失敗が隠蔽される組織文化
```
