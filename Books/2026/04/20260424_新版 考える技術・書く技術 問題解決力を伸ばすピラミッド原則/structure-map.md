# 構造マップ — 新版 考える技術・書く技術（ピラミッド原則）

| 部・章 | 機能 | 重要度 |
|--------|------|--------|
| 序文 | ミントの経歴と本書4部構成の案内 | 中 |
| 第Ⅰ部 書く技術（第1〜5章） | ピラミッド原理・導入部・演繹／帰納 | 最高 |
| 第Ⅱ部 考える技術（第6〜7章） | 論理順序・グループ要約・白紙主張の排除 | 高 |
| 第Ⅲ部 問題解決（第8〜9章） | R1/R2・診断フレーム・ロジックツリー・課題分析批判 | 高 |
| 第Ⅳ部 表現（第10〜11章） | 見出し・箇条書き・イメージ言語化 | 中 |
| 追補A/B | 不明推測法・要点一覧・監修／訳者あとがき | 低〜中 |

## 章別一行サマリー

1. **第1章**: 読み手は主メッセージから入りたい—時系列ではなくピラミッドへ再配列せよ。  
2. **第2章**: 縦関係（要約）と横関係（論理）と導入部のストーリー。  
3. **第3章**: トップダウン／ボトムアップの構築手順と初心者の注意。  
4. **第4章**: SCQA（状況・複雑化・疑問・回答）で導入部を設計する。  
5. **第5章**: 演繹と帰納の使い分けとリスク。  
6. **第6章**: 時間・構造・度合いの順序でグループ内を並べる。  
7. **第7章**: 箇条書きの上の要約を「行動の結果」まで落とす。  
8. **第8章**: 問題＝R1とR2のギャップ、オープニングと懸念の配置。  
9. **第9章**: 診断フレームとイエスノー質問、ロジックツリーで解決策を展開。  
10. **第10章**: 見出し階層・インデント・移行で構造を視覚化。  
11. **第11章**: イメージを言葉に写す／読み手の脳内イメージに合わせる。

## 概念の階層（最小）

```
主メッセージ（頂点）
 ├─ キーライン（主要グループ要約）
 │   ├─ 下位グループ（根拠・手順・事例）
 │   └─ ...
 └─ （導入部：状況→複雑化→疑問→キーライン提示）
```

## よく使う略号

| 略号 | 意味 |
|------|------|
| SCQA | Situation / Complication / Question / Answer |
| R1/R2 | 望ましくない結果 / 望ましい結果 |
| MECE | 漏れなくダブりなく（本章では分類の厳密さとして繰り返し登場） |
