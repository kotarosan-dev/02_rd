# structure-map

## 1. 本書の骨格

本書は「生成AIの個別モデル史」ではなく、「原論文ベースで基礎概念を積み上げる構造」を採る。
前半は読む技法と入力表現で土台を作る。
中盤はTransformerとGPTでテキスト生成の中心軸を作る。
その後に拡散モデルとCLIP系で画像・マルチモーダルへ広げる。
終盤はスケーリング則と評価で、研究開発の意思決定に接続する。
したがって依存関係は「論文読解 -> 表現学習 -> アーキテクチャ -> 大規模化 -> 推論強化 -> 評価」である。

## 2. 章間依存


## 4. 中心となる問い

- 第1章: 論文のどこまで読めば、主張の成立条件を見抜けるか。
- 第2章: 離散記号であるテキストを、学習しやすいベクトル表現へどう落とすか。
- 第3章: なぜSelf-Attentionが系列モデルの中心になったのか。
- 第4章: GPTはなぜ少数例提示だけで多タスクに対応できるようになったのか。
- 第5章: ノイズ除去の枠組みが、なぜ画像生成で強かったのか。
- 第6章: 画像とテキストを、どこでどう結びつければよいのか。
- 第7章: 大きくすることと、長く考えさせることは、どう違うのか。
- 第8章: 生成AIの「賢さ」は何で測ればよいのか。

## 5. 概念階層マップ

### レベル0: 読み方

- 論文は結論文より成立条件を読む。
- 数式は丸暗記でなく具体例で追う。
- 実装があるなら実装差分を見る。
- 評価指標の設計意図を読む。

### レベル1: 入力表現

- one-hot
- 埋め込み
- 分布仮説
- 単語分割
- サブワード分割
- バイト分割
- トークナイザー不要論

### レベル2: 計算構造

- Encoder / Decoder
- Self-Attention
- Query / Key / Value
- Scaled Dot-Product
- Multi-Head Attention
- Positional Encoding
- Residual + LayerNorm + FFN
- RoPE
- MoE

### レベル3: 学習目的

- 次トークン予測
- 半教師あり事前学習
- ゼロショット / one-shot / few-shot
- 対照学習
- 拡散のノイズ予測
- SFT
- RLHF
- GRPO

### レベル4: 生成対象

- テキスト
- 画像
- text-to-image
- text+image-to-image
- text+image-to-text

### レベル5: 評価

- BLEU
- FID
- ゼロショット精度
- 人間相対評価
- 数学・科学・コードベンチ
- 超高難度専門家ベンチ

## 11. 重要な対比

| 対比 | 本書の示す見方 |
|---|---|
| 単語 vs サブワード | 未知語対応と語彙爆発のトレードオフ |
| RNN vs Transformer | 逐次処理 vs 並列処理 |
| U-Net拡散 vs DiT | 畳み込み中心 vs Transformer中心 |
| 事前学習 vs 推論時強化 | 重みへの知識注入 vs その場での思考延長 |
| 人間評価 vs 専門ベンチ | 嗜好・自然さ vs 厳密正誤 |
| オープン論文 vs 非公開モデル | 原理解明のしやすさ vs 能力報告中心 |

## 12. 実務での使い回し順

1. 新しいモデルや論文を見たら、まず第1章の読み方で主張の成立条件を確認する。
2. 入力品質や未知語で問題があるなら、第2章でトークン化と埋め込みの選択肢に戻る。
3. 長文・速度・構造的な挙動を疑うなら、第3章で注意機構と位置表現に戻る。
4. 指示追従や対話品質を疑うなら、第4章で事前学習、SFT、RLHFの分担に戻る。
5. 画像品質や条件忠実性を疑うなら、第5章と第6章で拡散とマルチモーダル融合を見直す。
6. 数学・コード・長考タスクを強くしたいなら、第7章で推論時スケーリングと強化学習を見る。
7. 採用可否を決める段階では、第8章で評価指標の意味と限界を確認する。
