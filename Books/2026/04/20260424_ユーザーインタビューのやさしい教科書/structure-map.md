## 章別一行サマリー

- **はじめに**: 定性の強みと制約を理解し、日常会話との違い（目的・準備）を腹落ちさせる導入。
- **Chapter 1**: 調査目的→参加者像→計画書→インタビューガイドの順で、半構造化の骨格を固定する。
- **Chapter 2**: スクリーナーとWebアンケート運用、会場レイアウト、オンライン接続・同意、マイクと感染対策の両立。
- **Chapter 3**: ラポール、依頼主同席のリスク、質問の短さ／オープン・クローズ、深掘りと「間」、メタ認知。
- **Chapter 4**: タイプ別にデータを束ね、速報と図解、生の声と写真で開発サイドに返す。
- **Appendix**: 実務テンプレの型をそのまま借用し、初回プロジェクトのブートストラップに使う。

## 重要概念索引

| 用語 | 近い誤解 | 本書での正しい位置づけ |
|------|-----------|-------------------------|
| インタビュアー／イー | 聞き手／聞き手の混淆 | 聞き手＝モデレーター、相手＝ユーザーで統一 |
| 参加者（オンライン） | 観察者を含む総称 | 本書ではユーザー＝参加者、観察者は別枠 |
| 半構造化 | 無秩序に聞くこと | ガイドはあるが臨場で深掘り軌道を変える |

## データの流れ（調査〜開発）

```
計画（目的・属性・ガイド） → リクルート（スクリーナー・日程） → 実施（ラポール・深掘り）
    → 記録（音声・メモ） → 考察（タイプ別束ね） → 共有（速報・図・引用）
```

## 優先度の目安（実務）

1. 録音品質と同意・プライバシー（失うと全損）
2. 質問の誘導排除（データ歪み防止）
3. 依頼主の同席ルール（ラポール維持）
4. オンライン接続テストと代替日程
5. 報告の速さと具体性（現場の学習ループ）
