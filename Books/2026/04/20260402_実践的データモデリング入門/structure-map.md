# structure-map

## 1. Book Skeleton
- タイトル: 実践的データモデリング入門
- ジャンル判定: G
- 副次判定: C寄り
- 中心テーマ: 業務上の事実を一箇所で管理し、変更時にも説明できるモデルを作ること
- ケーススタディ: DBMe書店
- 構成: 基礎編 + 実践編 + おわりに

## 2. Macro Flow
| フェーズ | 主質問 | 主成果物 |
|---|---|---|
| 方向づけ | なぜモデリングするのか | 3層の理解、対象範囲 |
| 基本文法 | どう読めるモデルにするのか | 記法理解、キー・関係の定義 |
| 構造抽出 | 何を管理対象として切り出すか | エンティティ候補、主要属性 |
| 現場照合 | 実際の帳票や処理にどう現れるか | 正規化結果、CRUD表 |
| 統合 | 理想と現実をどう融合するか | 論理モデル |
| 実装 | DBMS上でどう動かすか | 物理モデル、索引、制約 |
| 運用 | 変更にどう耐えるか | 辞書、レビュー、版管理 |

## 3. Chapter Dependency
### まえがき
- 入力: なぜこの本を読むのかという問題意識
- 出力: データモデルを意味モデルとして読む視点
- 次章への橋: 第1章で3層構造と目的意識へ入る

### 第1章
- 入力: モデル化以前の混沌
- 出力: 概念・論理・物理の3層
- 次章への橋: 第2章で記法の基本文法を揃える

### 第2章
- 入力: モデルの骨格
- 出力: エンティティ、属性、関係、キーの読み方
- 次章への橋: 第3章でデータモデルの限界を補う

### 第3章
- 入力: データだけでは業務が見えないという問題
- 出力: IDEF0、DFD、CRUD、UML の位置づけ
- 次章への橋: 第4章でトップダウン抽出へ進む

### 第4章
- 入力: 業務をどう切るかという問い
- 出力: 資源・出来事・集約の見分け方
- 次章への橋: 第5章で主キーと主要属性を締める

### 第5章
- 入力: エンティティ候補
- 出力: 主キー候補、主要属性
- 次章への橋: 第6章で名前と型の標準へ進む

### 第6章
- 入力: 複数人で共有されるモデル

- 次章への橋: 第7章で現場資料から逆算する

### 第7章
- 入力: 既存帳票・画面・ファイル

### 第8章

### 第10章

- 出力: 制約、参照整合性、ビジネスルール
- 次章への橋: 第11章で定石パターンへ広げる

### 第11章

### 第13章

- 要約時は、図法の細部よりも判断理由と章間接続を優先した。
- 第9章、第13章、第17章を背骨として再配置した。
- 物理実装章は個別テクニックの羅列にせず、論理意味を守るという観点で束ねた。
