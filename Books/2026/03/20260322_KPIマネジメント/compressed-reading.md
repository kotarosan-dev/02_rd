## 読書ナビゲーション

本書は「数字をたくさん見ること」をKPIだと思っている組織を、**KGI・CSF・KPIの一直線の構造**へ引き戻す本です。著者の主張は一貫していて、KPIとは「事業成功の鍵を数値目標にしたもの」であり、単なるダッシュボードではありません。だから出発点はKPIではなく、まず「何を成功と呼ぶのか」というKGIの確認であり、次にそれを生むプロセスのモデル化であり、そこから最重要プロセスを1つに絞ることになります。

読む順番としては、初学者は第1章で定義と8ステップ、第2章で「なぜ1つなのか」とPDDS、第5章で設計ワークを見ると骨格がつかめます。実務者は第4章の事例集を参照しつつ、自分の現場を「量・CVR・単価・費用」のどこで改善するのかへ置き換えると使いやすいです。著者が何度も言う通り、KPIは管理の飾りではなく、**現場が打ち手を選ぶための信号**です。

> 「ＫＰＩとは、『事業成功』の『鍵』を『数値目標』で表したもの。」

> 「ＫＰＩは『信号』だから『１つ』。」

## コアフレームワーク

### 1. 本書の中心式

```text
KGI（最終目標）
  ↓ 何がゴールかを明確化
現状とのギャップ確認
  ↓ このままでは足りない分を見える化
事業プロセスのモデル化
  ↓ 売上 = 量 × CVR × 単価 × 手数料率 などに分解
CSF（最重要プロセス）を1つに絞る
  ↓ 現場がコントロールできる先行指標を選ぶ
KPI（CSFの数値目標）を置く
  ↓ 整合性・安定性・単純性を確認
悪化時の対策を事前合意
  ↓ PDDSで回す
組織知として横展開
```

### 2. 主要概念の最短定義

| 概念 | 本書での意味 | 外しやすいポイント |
|------|--------------|--------------------|
| `KGI` | 期末に達成したい最終目標数値 | 売上・利益・ユーザ数など、そもそものゴールが曖昧 |
| `CSF` | KGI達成のための最重要プロセス | 重要そうなものを複数残してしまう |
| `KPI` | CSFをどの程度やればKGIに届くかを示す数値目標 | ただの管理指標や遅行指標を置いてしまう |
| `PDDS` | Plan-Decide-Do-See の改善サイクル | 実行だけ増えて振り返りがない |
| `TTPS` | 徹底的にパクって進化させる | 個人の工夫が組織知へ昇華されない |

### 3. KPI設計の10ステップ

1. KGIを確認する。
2. 現状とのギャップを確認する。
3. 事業プロセスをモデル化する。
4. CSFを1つに絞る。
5. CSFの目標数値としてKPIを置く。
6. `整合性・安定性・単純性` を確認する。
7. 悪化時の対策を事前検討する。
8. 関係者のコンセンサスを取る。
9. 運用する。
10. 継続的に改善する。

### 4. 著者が重視する3つの設計原則

| 原則 | 意味 | 実務上の含意 |
|------|------|--------------|
| 整合性 | KPIが動けば本当にKGIが動くか | 相関だけでなく因果と現場感を確認する |
| 安定性 | リアルタイムに近い形で継続取得できるか | 外部依存・集計負荷・遅い統計を避ける |
| 単純性 | 現場が理解し、覚え、動けるか | 数式は正しくても複雑すぎるものは失敗する |

## 章別エッセンス

### はじめに

**核心命題**: KPIマネジメントは「数字を見ること」ではなく、`CSFを共有し、KPIを実行し、改善し続けること` である。  
**反直感的洞察**: KPIの本質はツールではなく、現場の判断様式にある。  
**事例サマリー**: リクルートで11年続いた社内講座と実際の事業責任経験が本書の土台。  
**行動指針**:

□ 自組織のKPIを「知っているか」ではなく「使っているか」を問い直す  
□ `KGI / CSF / KPI` を混同していないか確認する

### 第1章 KPIの基礎知識

**核心命題**: KPIは「事業成功の鍵」の数値目標であり、設計手順を踏まないと必ず壊れる。  
**反直感的洞察**: ダメなKPIの多くは、分析不足ではなく「順番の誤り」から生まれる。  
**事例サマリー**: GDPのように相関があっても現場で制御できず、取得も遅い指標はKPIにならない。  
**行動指針**:

□ 最初にKGIとその数値を関係者で合意する  
□ 事業を数式化し、定数と変数を分ける  
□ KPI悪化時の `時期・程度・施策・決裁者` を事前に決める

### コラム1 前からやるか、後ろから考えるか

**核心命題**: 目の前の作業から始める人より、目的・データの確からしさ・最後の使い道から考える人のほうが生産性が高い。  
**反直感的洞察**: 着手が早いことは、しばしば要領のよさではなく事故の前触れである。  
**事例サマリー**: 「何のために分析するのか」「データは正しいか」を先に問うだけで無駄仕事が減る。

### 第2章 KPIマネジメントを実践するコツ

**核心命題**: KPIは1つであるべきで、全員が見られる信号として運用されるべきである。  
**反直感的洞察**: KPIを増やすことは管理精度を上げるのではなく、振り返り不能な組織を作る。  
**事例サマリー**: 分母が変数の比率KPIは現場をゆがめる。提案率より提案数、打率よりヒット数を追う発想。  
**行動指針**:

□ KPIは現場が即時に理解できる1指標へ寄せる  
□ 覚えやすい数字に丸める  
□ `バカの壁` と `不安の壁` を越え、絞り込みを恐れない  
□ PDDSでは `See -> Plan` の学習接続を最重視する

### コラム2 TTPS

**核心命題**: 振り返りで得た知恵は、個人の工夫で終わらせず、他拠点が徹底的に真似して進化させると組織知になる。  
**反直感的洞察**: 小さな拠点でも、一次創造者でなく二次改良者になることで強い発信源になれる。  
**事例サマリー**: 福岡天神店が他店施策を高速でTTPSし、少人数拠点からナレッジの供給源へ変わった。

### 第3章 実践前に知るべき3つのこと

**核心命題**: KPI設定の前に、会社の方向性・KGIの本質・利益構造を理解しておかないと、部分最適なKPIになる。  
**反直感的洞察**: KGIの本質は売上ではなく、ゴーイングコンサーンを可能にする継続利益にある。  
**事例サマリー**: 年頭所感や戦略資料から「重要語」を抜き出し、現場のKPIと接続する。費用と売上が絡む項目は定数化で判断速度を上げる。  
**行動指針**:

□ まず会社の方針文書を読む  
□ 利益 = 売上 - 費用 を起点に、相互影響する変数を見抜く  
□ 毎回悩む項目は平均や上限を決めて定数化する

### 第4章 KPI事例集

**核心命題**: KPI設計は業種が違っても、`構造分解 -> ボトルネック特定 -> 1指標化` の型で再利用できる。  
**反直感的洞察**: 売上を伸ばしたいなら、まず量を増やす前に内部プロセスと歩留まりを改善したほうが総費用が小さい。  
**事例サマリー**:

| 領域 | 1行要約 |
|------|----------|
| 営業活動 | 量・CVR・単価の三択で考え、まず受注率やプロセス短縮から着手する |
| エリア戦略 | 狭い商圏・特定顧客群に絞ると学習が速くなる |
| 重点顧客攻略 | 高収益顧客の共通点とライトパーソン接触をCSF化する |
| 商品シフト | 戦略転換は本番前に特定エリアでトライアルし、営業・制作・検証のノウハウを先に取る |
| 従量課金 | まず手数料率と歩留まり、次にリーチ。行動シェアを上げる |
| 採用 | 重要KPIは応募数ではなく採用選考スピードになりやすい |
| 広報 | 目的を採用・販促・ブランドのどれかに絞るとKPI化できる |
| 社内スタッフ | 従業員満足度に閾値を置き、資源配分の信号にする |
| 集客 | 集客担当にもマッチング目標を持たせ、集客単価の範囲で裁量を渡す |
| 個人成長 | 読書量や学習量もKPI化できる |
| 健康 | 歩数や歩行時間のように、健康寿命に効く行動をKPIに置く |

**行動指針**:

□ まず「量を増やす」より「歩留まりを上げる」を疑う  
□ 施策単位で時間短縮を測る  
□ 大口顧客や売れ筋商品もABC/ABM的に利益で見直す

### 第5章 KPIを作ってみよう

**核心命題**: KPIは概念理解だけでは定着せず、設計書・役割・合意・対策まで書いて初めて運用可能になる。  
**反直感的洞察**: 多くの失敗は「KPIそのもの」ではなく、運用前の準備不足で起きる。  
**事例サマリー**: 会社、商品、ダイエットの3ケースで、対象・目的・期間・関係者・KGI・ギャップ・CSF・KPI・悪化時対策までを設計書化。  
**行動指針**:

□ `対象・目的・期間` を最初に書く  
□ `最終決裁者・承認者・事務局` を曖昧にしない  
□ KPI未達時の分岐を事前に文書化する  
□ KPI達成とKGI達成がずれたら、構造か水準のどちらが誤っているか振り返る

### おわりに

**核心命題**: KPIは著者にとって理論ではなく、リクルートで鍛えた講義と事業運営の実践知である。  
**事例サマリー**: 執筆の背景には社内講座、転職、周囲の信頼、実務での再検証があった。  
**意味**: 本書は抽象論より「実地の型」を渡すことを狙っている。

## 批判的視座

本書の強さは、KPIを飾りの数値から、現場が打ち手を決めるための「一本化された信号」へ戻している点にあります。特に、`コントロール可能性`、`取得の速さ`、`現場の理解可能性` を同時に問うのは、KPI実務で非常に有効です。一方で、限界もあります。

第一に、本書は「最重要プロセスは1つに絞れる」前提を強く置きます。これは立ち上げ期や単一ボトルネックがある事業には効きますが、複数制約が同時に競合する複雑系の事業や、品質・安全・収益を同時管理すべき現場では、そのまま適用しにくい。第二に、著者はKPIの設計と運用には強い一方、KPIが評価・査定・権限配分と結びついたときのゲーム化や政治性にはあまり踏み込みません。第三に、創造性、学習、信頼、疲弊のような測りにくい価値は、KPI一本化の外側へ追いやられやすい。したがって本書は、**KPIをやめるための本ではなく、雑なKPI運用を卒業するための本**として読むのが最適です。

## 実践チェックリスト

□ いま見ている数字はKGIか、CSFか、KPIかを言い分けられる  
□ ゴールそのものと目標数値について関係者合意がある  
□ 事業を `量 × CVR × 単価 × 費用` のように式で表現できる  
□ 変数と定数を分け、現場で動かせる変数に絞れている  
□ KPIは1つで、先行指標で、現場が制御可能である  
□ KPI悪化時の `時期・閾値・施策・決裁者` を先に決めている  
□ 振り返り会議を施策承認時点で同時設定している  
□ 成功事例も失敗事例もTTPSできる形で残している  
□ KPI達成なのにKGI未達、またはその逆が起きたときの見直し手順がある

## 原著への架け橋

この圧縮読本では、原著の事例細部と著者の語り口をかなり削っています。原著の価値は、フレームワークそのもの以上に、「どこで現場が反発するか」「なぜ複数KPIが振り返りを壊すのか」「どの順番で改善すると組織が動くのか」を、営業・採用・広報・健康まで横断して具体例で体感させるところにあります。特に第4章のケース集は、自分の業種へ置き換えるヒントの宝庫です。

また、著者の口調には、数字を愛好しつつも数字の粗雑な扱いを嫌う独特の実務感覚があります。その温度感は圧縮では完全には残りません。自組織で本気でKPI設計書を作る、あるいは「なぜ1つに絞るのか」を周囲に説明する必要がある人は、原著に戻る価値があります。
