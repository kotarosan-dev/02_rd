## 読書ナビゲーション

本書は「どのツールで描くか」より前に、「何を、なぜ、その見せ方で示すのか」を鍛える本です。中心にあるのは、**データを正確に伝えること**と**読み手の知覚負荷を下げること**の両立で、前半はチャート選択、後半はデザイン原則、最後にファイル形式やストーリーテリングまで広げます。コード例がほぼ出てこないのは、ツール依存の小手先ではなく、長く残る判断基準を渡すためです。

技術書として読むときのコツは、全章を同じ重みで追わないことです。まず「今の悩み」に対応する章から入り、必要になったときに周辺章へ戻ると吸収しやすい構造になっています。チャート選定に悩むなら第5章と第6〜16章、見栄えと読みやすさの改善なら第17〜25章、資料作成や配布まで含めて整えるなら第27〜29章が効きます。

> 「コードではなく、考え方と図に焦点を当ててほしい」

> 「データの可視化は、芸術と科学にまたがる作業です」

## 概念マップ

```text
データ
  -> 何を伝えたいかを定める
  -> 量 / 分布 / 内訳 / 関連 / 時系列 / 地理 / 不確かさ を選ぶ
  -> 適切なエスセティクスにマッピングする
       位置 / 長さ / 面積 / 色 / 形 / サイズ / 線種
  -> 読み手の知覚に合わせてデザインする
       比例 / 比較しやすさ / 色覚多様性 / 凡例負荷 / 文脈
  -> ストーリーとして配置する
       何を強調するか / どこを削るか / どう記憶に残すか
```

### 本書の4本柱

| 柱 | 中心質問 | 該当章 |
|----|----------|--------|
| 1. マッピング | どの変数をどの視覚要素で表すか | 2〜4 |
| 2. チャート選択 | 何を伝えたいときに何を使うか | 5〜16 |
| 3. デザイン原則 | 見やすく、誤読されず、比較しやすいか | 17〜25 |
| 4. 実務運用 | 保存形式、ツール選定、物語化をどうするか | 27〜29 |

### まず覚える判断軸

| 問い | 判断基準 |
|------|----------|
| 量を見せたいか | 棒ではなくドットでよい場面が多い。ゼロ起点が守れないなら棒を疑う |
| 分布を見せたいか | ヒストグラムはビン幅、密度曲線はバンド幅に依存する |
| 比率を見せたいか | 線形軸では0起点、対数軸では1起点 |
| 比較したいか | 共有ベースラインがあるか、位置で比較できるか |
| 色を使うべきか | 区別・値・強調のどの用途かを先に決める |
| 3Dにすべきか | ほぼ不要。真に3D対象を扱うときだけ慎重に使う |

## 章別エッセンス

### 第1章 はじめに

**核心命題**: よい可視化は「正確さ」と「美しさ」の両方を満たす。  
**重要語**: `ugly` は見栄えが悪い、`bad` は知覚上不適切、`wrong` は数学的に誤り。  
**行動指針**: 「嫌な見た目」も「誤解を生む見た目」も別物として判定する。

### 第2章 データからエスセティクスへのマッピング

**核心命題**: 可視化は、データ値を位置・形・大きさ・色・線種へ写像する作業。  
**What**: 連続値に強いのは位置・長さ・色・太さ。形や線種は主に離散値向き。  
**Why**: ここを誤ると、後でどれだけ装飾しても本質が崩れる。  
**When**: どのチャートにするか決める前。

### 第3章 座標系と座標軸

**核心命題**: 座標系はただの器ではなく、読み手の比較方法を決める。  
**要点**: 線形軸、対数軸、平方根軸、極座標、地図投影を使い分ける。  
**判断基準**: 比率なら対数、面積的広がりなら平方根が効く場面がある。周期性には極座標が効くが乱用はしない。

### 第4章 カラースケール

**核心命題**: 色には3用途しかない。区別、値の表現、強調。  
**判断基準表**:

| 用途 | 使う色 |
|------|--------|
| 項目を区別 | 質的スケール |
| 量を表す | 連続的スケール |
| 中立点からの偏差 | 発散的スケール |
| 一部だけ目立たせる | アクセントカラー |

**注意**: 色は万能ではなく、細かい数量比較には弱い。

### 第5章 ビジュアライゼーション概説

**核心命題**: チャートの百科事典ではなく、「どんな問いにどんな図が向くか」の索引。  
**効きどころ**: 量、分布、内訳、関係、時系列、地図、不確かさを大づかみで見渡せる。  
**使い方**: 迷ったらここに戻って、章を選び直す。

### 第6章 量を可視化する

**核心命題**: 棒グラフは便利だが万能ではない。  
**What**: 棒グラフ、グループ化、積み上げ、ドットプロット、ヒートマップ。  
**Why**: 棒はゼロ起点が必須で、比較の焦点が「端点」ではなく「長さ」になる。  
**When**: カテゴリごとの量を見せたいとき。  
**実務判断**: 差分を見せたいだけならドットプロットのほうが有利な場面が多い。

### 第7章 ヒストグラムと密度プロット

**核心命題**: 分布可視化はパラメータ選びが結果を大きく変える。  
**パターン**: ヒストグラムはビン幅、密度曲線はバンド幅とカーネルが効く。  
**アンチパターン**: 密度推定が存在しえない値まで裾を伸ばしていないか確認しない。  
**結論**: 複数分布の重ね合わせでは、ヒストグラムより密度曲線が機能しやすい。

### 第8章 ECDFとQ-Qプロット

**核心命題**: 分布を恣意パラメータなしで見たいならECDFとQ-Q。  
**What**: ECDFは累積構造、Q-Qは基準分布からのズレを見る。  
**Why**: ヒストグラムや密度曲線の恣意性を減らせる。  
**When**: 歪みの大きい分布や、正規性・対数正規性の確認。

### 第9章 たくさんの分布を一度に可視化する

**核心命題**: 分布が増えるほど、平均とエラーバーだけでは足りない。  
**主要手段**: 箱ひげ図、バイオリンプロット、ストリップチャート、シーナプロット、リッジライン。  
**判断基準**: データ点が少ないなら生データ、十分多いならバイオリンやリッジライン。  
**注意**: エラーバーはばらつきか不確かさかを誤読されやすい。

### 第10章 内訳を可視化する

**核心命題**: 円グラフは常に悪ではないが、比較タスクに強いわけでもない。  
**使い分け**:

| 見せたいこと | 向く図 |
|--------------|--------|
| 全体に対する単純な割合 | 円グラフ |
| 時系列や複数条件の変化 | 積み上げ棒 |
| 各項目の大小比較 | 横並び棒 |
| 連続変数に応じた構成変化 | 積み上げ密度 |

### 第11章 入れ子になった内訳

**核心命題**: カテゴリが複数階層になると、単純な円グラフや棒では誤解が起きる。  
**What**: モザイク図、ツリーマップ、入れ子円、パラレルセット。  
**Why**: 二重カウントや階層関係の不可視化を避ける。  
**When**: 複数カテゴリの組み合わせを一気に見たいとき。  
**制約**: 比較のしやすさは必ず落ちる。数値ラベルで補助したほうがよい。

### 第12章 複数の量的変数間の関連性

**核心命題**: まず散布図、必要なら抽象化。  
**流れ**: 散布図 -> バブルや散布図行列 -> コレログラム -> PCAなど次元削減。  
**重要洞察**: バブルはサイズ知覚が弱い。多変量なら散布図行列や主成分空間のほうが読めることが多い。  
**対応データ**: `x = y` 線やスロープグラフが効く。

### 第13章 時系列など独立変数の関数

**核心命題**: 順序があるなら線でつなぐ価値が生まれる。  
**要点**: 点を結ぶ線はデータそのものではなく視線誘導。複数系列なら直接ラベルが凡例より強い。  
**特殊形**: 用量反応曲線、連結散布図。  
**判断基準**: 2変数の相互運動や循環を見たいなら連結散布図。

### 第14章 トレンドを可視化する

**核心命題**: 平滑化やフィッティングは便利だが、どれも「解釈」であって真実そのものではない。  
**What**: 移動平均、LOESS、スプライン、明示的な関数、対数変換、デトレンド、分解。  
**Why**: ノイズを減らし、長期トレンドや周期成分を見抜く。  
**When**: 生データのままだと傾向が埋もれるとき。  
**重要判断**: 非線形を直接当てるより、変換して線形で見るほうが安全なことが多い。

### 第15章 地理空間データ

**核心命題**: 地図は直感的だが、投影と面積効果を理解していないと危険。  
**主要論点**: 投影法、レイヤー、コロプレス、カルトグラム。  
**判断基準**: 密度を塗るならコロプレスは強い。絶対量や州サイズ差が大きいと誤導しやすい。  
**アンチパターン**: アラスカの大きさに目を奪われる全米州地図。

### 第16章 不確かさを可視化する

**核心命題**: 不確かさは見えないが、見せないと過信を生む。  
**What**: エラーバー、信頼帯、分位点ドット、事後分布、HOP。  
**Why**: 推定値が一点で確定しているかのような誤読を防ぐ。  
**When**: 平均、差、回帰曲線、予測を示すとき。  
**最大の注意**: 標準偏差と標準誤差を混同しない。

### 第17章 プロポーショナルインクの原則

**核心命題**: 色を塗った面積は、表す値に比例していなければならない。  
**実務ルール**:

| 状況 | 原則 |
|------|------|
| 線形軸の棒 | 0起点 |
| 対数軸で比率を見る棒 | 1起点 |
| 塗りつぶし領域 | 面積が値に比例 |
| 途中から切った棒 | 原則アウト |

**この章の破壊力**: よくある「見た目は派手だが差を誇張する図」を一掃できる。

### 第18章 重なり合った点

**核心命題**: 点が重なり始めたら、散布図の読み方自体を変える。  
**手段**: 半透明化、ジッタリング、2次元ヒストグラム、六角形ビン、等値線。  
**判断基準**: 少し重なるなら透明化とジッタ、多すぎるならビニング、群ごとの差なら等値線を分割表示。  
**アンチパターン**: 色分け等値線を一枚に重ねすぎて毛玉にする。

### 第19章 色の落とし穴

**核心命題**: 色は、よかれと思って使うほど壊しやすい。  
**3大失敗**:

| 失敗 | 何が起こるか |
|------|--------------|
| 色数が多すぎる | 凡例照合で疲れる |
| 意味のないレインボー | 視覚的ノイズだけ増える |
| 非単調スケール | 値の大小関係を壊す |

**追加論点**: 色覚多様性を考慮しない設計は、図を一部の人に読めなくする。

### 第20章 表現の冗長化

**核心命題**: 色だけに頼るな。  
**What**: 形、線種、直接ラベル、軸に埋め込んだ凡例。  
**Why**: 色覚差、印刷、凡例照合の負荷に耐える。  
**When**: グループ比較、折れ線複数本、散布図群。  
**結論**: 可能なら凡例を消し、図の中に直接意味を書き込む。

### 第21章 マルチパネル図

**核心命題**: パネルを増やすほど、軸の統一と並び順が内容そのものになる。  
**要点**: スモールマルチプルは同一比較のため、複合図は異なる問いの接続のため。  
**実務ルール**: 軸範囲は揃える。意味ある順序で並べる。ラベルは控えめに、一貫した視覚言語で。

### 第22章 タイトル、キャプション、表

**核心命題**: 図は絵ではなく、情報伝達の単位。  
**要点**: タイトルは図の要点を言い切る。軸タイトルは単位まで書く。表は縦線を避け、数値列は右揃え。  
**実務判断**: 図のキャプションは下、表のキャプションは上。

### 第23章 データとコンテキスト

**核心命題**: データ以外のインクは減らすべきだが、ゼロにはしない。  
**What**: 背景、枠、グリッド、対角線、基準線。  
**Why**: 比較軸と読み方のガイドを与えるため。  
**When**: 対応データ、株価、スモールマルチプルなど。  
**判断基準**: 重要な変化に直交する方向へグリッドを置く。

### 第24章 軸ラベルをもっと大きくする

**核心命題**: ほとんどの図は、テキストが小さすぎる。  
**重要ルール**: 最終表示サイズに縮小して確認する。拡大表示だけで判断しない。  
**実務効果**: 伝達力のわりに、最も費用対効果が高い改善の一つ。

### 第25章 線画を避けよう

**核心命題**: 輪郭線より塗りつぶしのほうが、人はまとまりとして知覚しやすい。  
**適用先**: ヒストグラム、密度曲線、散布図記号、箱ひげ図。  
**判断基準**: 白抜き・破線・輪郭だらけは視覚ノイズを増やしやすい。  
**例外**: プロポーショナルインク原則を壊してまで塗りつぶさない。

### 第26章 3Dに走らない

**核心命題**: 3Dはたいてい装飾であり、比較を壊す。  
**What**: 余計な3D、3D位置スケール、例外的に許される3D。  
**Why**: 2D投影で奥行き情報が失われ、手前が大きく見える。  
**When**: 真に3Dな対象を、回転や対話操作つきで見せるときだけ。

### 第27章 画像ファイル形式

**核心命題**: 図の品質は、描いた後の保存形式でも壊れる。  
**結論**:

| 用途 | 推奨 |
|------|------|
| 印刷や高品質保存 | PDF |
| Webや線画 | PNG |
| 写真中心 | JPEGを最終手段で |

**禁止寄り**: 線画や文字入り図をJPEGで保存しない。

### 第28章 可視化ソフトウェアを選ぶ

**核心命題**: 最良のソフトは、自分に必要な図を再現可能に作れるソフト。  
**評価軸**: 再現性、反復性、探索の速さ、表現の自由度、コンテンツとデザインの分離。  
**本音**: 対話的ツールは探索には便利でも、再生成性で弱くなりがち。

### 第29章 ストーリーを伝え、強調する

**核心命題**: 可視化は説明ではなく、物語の一部。  
**重要フレーズ**: 「司令官に伝わる図を作る」。  
**意味**: 賢い読者でも、忙しい読者は複雑図を解読してくれない。  
**実務ルール**: まず単純な図で問いを立て、必要なら徐々に複雑化する。反復は避け、一貫性は保つ。

## 判断基準表

| 状況 | 選ぶ | 避ける |
|------|------|--------|
| カテゴリごとの単純な量 | 棒 or ドット | ゼロ起点でない棒 |
| 複数分布の比較 | 密度、バイオリン、リッジライン | 積み上げヒストグラム |
| 2変数の関係 | 散布図 | いきなり3D散布図 |
| 絶対量の地理比較 | カルトグラムや別表現も検討 | 州サイズ差の大きいコロプレス |
| 推定の不確かさ | エラーバーや信頼帯を明記 | 説明なしのエラーバー |
| たくさんの点 | 透明化、ビニング、等値線 | 色数だけ増やす散布図 |
| 相手が非専門家 | 単純図から導入 | 一枚に全情報を載せる |

## パターン／アンチパターン

| パターン（推奨） | アンチパターン（回避） | 理由 |
|------------------|--------------------------|------|
| 位置で比較する | 面積や3D遠近で比較させる | 人は位置比較に強い |
| 直接ラベル | 凡例を遠くに置く | 視線往復が減る |
| 最小限のグリッド | 過剰な枠線と濃い補助線 | データが主役になる |
| 変換して線形で見る | 非線形をそのまま重く当てる | 解釈しやすい |
| 色用途を限定する | レインボーで全部表す | 値関係が壊れる |
| 最終サイズで確認 | 作業中の拡大画面だけ見る | ラベルが小さくなりやすい |

## 落とし穴マップ

| 落とし穴 | 症状 | 回避策 |
|----------|------|--------|
| `ugly / bad / wrong` を混同 | 美観の話と数学誤りが曖昧 | 3分類で直す順序を決める |
| 棒の途中切り | 差が劇的に見えすぎる | 線形量は0起点、比率は1起点 |
| 密度推定の過信 | 負の年齢のような虚構が出る | 取り得ない値域を確認 |
| 色の使いすぎ | 凡例がパズルになる | 8カテゴリ超なら直接ラベル |
| 3D化 | 手前が大きく見える | 2Dの代替を先に探す |
| JPEG保存 | 文字や線がにじむ | PNG/PDFを使う |
| 忙しい相手に複雑図 | 何も伝わらない | 単純図 -> 複雑図の順にする |

## 批判的視座

本書の強みは、視覚化を「センス」ではなく**知覚の設計**として説明するところにあります。一方で、2019年原著ゆえにインタラクティブ可視化やダッシュボード運用、Web埋め込み、実装ライブラリごとの差異は薄めです。また、あえてコードを外しているため、実務導入では「原則は分かったが手が動かない」と感じる読者も出ます。つまり本書は**これ一冊で作れるようになる本**ではなく、**どんな図がダメで、どんな図が効くのかを判定できるようになる本**です。

さらに、著者はかなり明快に「避けるべき図」を切っていくため、例外場面では少し保守的に見えるかもしれません。たとえば円グラフや3Dに関しては、限定条件下での価値も認めつつ、実務ではまず疑ってかかる態度を取っています。この厳しさは、現場での誤用の多さを考えると妥当です。

## 実践チェックリスト

□ まず「何を伝えたいか」を量・分布・内訳・関係・時系列・地理・不確かさのどれかで言い直したか  
□ 値を位置・長さ・面積・色のどれで比較させているか意識したか  
□ 棒や面塗りがプロポーショナルインクの原則を破っていないか  
□ 色の用途を1つに絞れているか  
□ 凡例を直接ラベルへ置き換えられないか考えたか  
□ グリッドや背景がデータより目立っていないか  
□ ラベルを最終表示サイズで読めるか確認したか  
□ 図を見た相手が10秒で要点を言えるか想像したか  
□ 保存形式が用途に合っているか確認したか

## 原著への架け橋

この圧縮読本で残したのは、本書の**判断基準**と**禁止事項の骨格**です。原著でしか得られない価値は三つあります。第一に、各章で並ぶ大量の図版比較です。著者は同じデータを複数の描き方で見せることで、なぜそれが `ugly` で、`bad` で、`wrong` なのかを身体感覚に落とし込みます。第二に、各図のキャプションに含まれる細かな観察です。第三に、章をまたいで反復される「知覚と判断のしつこい接続」で、これが読後に図を見る目を変えます。

原著に戻る優先順位は、まず第5章を索引として使い、次に今困っている章へ飛ぶ読み方が効率的です。実務で最も元が取れるのは、第6〜10章、第15〜20章、第23〜25章、第29章です。
