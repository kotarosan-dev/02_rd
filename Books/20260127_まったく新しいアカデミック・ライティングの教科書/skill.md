# Academic Writing Framework

阿部幸大『まったく新しいアカデミック・ライティングの教科書』（光文社、2024）に基づくアカデミック・ライティングの実践フレームワーク。

## 概要

本フレームワークは、論文執筆を「アーギュメントの構築と論証」というコアプロセスに還元し、再現可能な技術として体系化する。人文系を中心に、社会科学・自然科学にも応用可能。

## 使用場面（When to Use）

- 学部生のレポート・卒業論文執筆
- 大学院生の修士論文・博士論文執筆
- 研究者の査読論文執筆
- 研究計画書・助成金申請書の作成
- 論文指導・ライティング教育

## コアコンセプト

### 1. 論文の定義
> 論文とは、アーギュメントを提出し、それが正しいことを論証する文章である。

### 2. アーギュメントとは
- **定義**: 論証が必要な主張（テーゼ）
- **条件**: 反論可能性を持つこと
- **形式**: 「AがBをVする」（他動詞モデル）

### 3. アカデミックな価値
- 価値は「面白さ」ではなく「引用と批判」によって創出される
- 既存の「学術的会話」に参入し、それを「更新」することで価値が生まれる

## フレームワーク/プロセス

### Phase 1: アーギュメント構築

#### Step 1: トピック→アーギュメントへの変換

| レベル | 例文 | 問題点 |
|--------|------|--------|
| トピック設定 | 「女性キャラクターに着目する」 | 論証を要求しない |
| 方法論宣言 | 「フェミニズム理論で分析する」 | 論証を要求しない |
| 価値判断 | 「ジェンダーが重要である」 | 曖昧すぎる |
| **アーギュメント** | 「男性中心主義的物語が女性キャラを排除する」 | ✓ 論証を要求する |

#### Step 2: 他動詞モデルでの強化

```
漠然とした観察 → 「AがBをVする」形式に変換

例:
「女性が蔑ろにされている」（受動態・曖昧）
↓
「男性中心主義的物語が女性キャラクターを排除している」（能動態・明確）
```

**チェックポイント:**
- [ ] 主体A（誰が/何が）は明確か
- [ ] 客体B（誰を/何を）は明確か
- [ ] 動詞V（どうする）は具体的か（have/do/make/beを避ける）
- [ ] 反論可能性があるか（「本当か？」と問える）

#### Step 3: アーギュメントの研磨

各要素を問い直す:
- **A（主体）について**: 具体的にはどのような事態を指すのか？
- **V（動詞）について**: 具体的にはどのような作用を指すのか？
- **B（客体）について**: その結果、何が起きるのか？

### Phase 2: アカデミックな価値の創出

#### Step 1: 先行研究の「会話」を把握する

```
1. 同様のトピックについて何が語られてきたか
2. 現状のコンセンサスは何か
3. どこに隙間や問題があるか
```

#### Step 2: 引用と批判のセット

```
「いまAだと思われているが、じつはBなのである」

- A = 先行研究の主張（引用）
- B = あなたのアーギュメント（主張）
- 移行部 = なぜAでは不十分か（批判）
```

**注意**: 批判対象が「ひろく受け入れられている」ほど価値は大きい

### Phase 3: パラグラフ・ライティング

#### 基本構造

```
1. Topic Sentence（主張）
   - パラグラフの中心的主張を1文で

2. Supporting Sentences（論証）
   - 主張を裏付ける証拠・論理
   - 具体例、データ、引用

3. Concluding Sentence（結論/移行）
   - パラグラフのまとめ
   - 次パラグラフへの橋渡し
```

#### パラグラフ間の論理

```
各パラグラフのTopic Sentenceだけを抜き出して読んでも
論文全体の流れが把握できるか？
→ できなければ構成を見直す
```

### Phase 4: イントロダクションとコンクルージョン

#### イントロダクション3点セット

```
1. 背景/文脈: なぜこのトピックが今重要か
2. 先行研究の問題点: 何が不足しているか
3. 本論のアーギュメント: 本論文は何を主張するか
```

#### コンクルージョンの役割

```
- 要約 ≠ 結論
- 結論 = アーギュメントの再確認 + 含意の展開
- 「だから何なのか」「何が変わるのか」を示す
```

## 具体的な実行手順

### 論文執筆のワークフロー

```
1. 【素材収集】
   トピックに関する資料を読む
   ↓
2. 【観察の記録】
   気になった点をメモする
   ↓
3. 【アーギュメント試作】
   観察を他動詞モデルで書いてみる
   ↓
4. 【先行研究との対話】
   既存研究を引用・批判できるか確認
   ↓
5. 【アーギュメント確定】
   論証可能性・反論可能性を検証
   ↓
6. 【構成設計】
   パラグラフ単位で論証の流れを設計
   ↓
7. 【執筆】
   パラグラフごとに書く
   ↓
8. 【推敲】
   Topic Sentenceの連続で論理を検証
```

## 入力/出力

### 入力
- トピック（研究対象）
- 先行研究の知見
- 一次資料・データ

### 出力
- アーギュメント（1文で表現可能な主張）
- 論文構成（パラグラフ単位の設計図）
- 完成原稿

## 注意点

1. **問いは必須ではない**: 問いをたてることは有用だが、本質ではない。問いがなくてもアーギュメントは成立する。

2. **面白さへの依存を避ける**: 「面白い」は主観的。アカデミックな価値は引用と批判で客観的に示す。

3. **受動態を避ける**: 「〜されている」ではなく「AがBをVする」と書く。

4. **強い動詞を使う**: have, do, make, be などの無内容な動詞を避け、具体的な動詞を選ぶ。

5. **最初から完璧を目指さない**: アーギュメントは繰り返し鍛えて強化する。
