# 購買メカニズム言語化フレームワーク

## 概要

本フレームワークは、消費者が「なぜモノを買うのか」を言語化し、マーケティング施策を設計するための体系的アプローチである。

### 購買の三層構造

人がモノを買う理由には3種類のメカニズムがある：

1. **なじみがあるから買う（流暢性）** - 接触頻度により想起集合に入る
2. **良いから買う（ポジショニング）** - 競合比較でPoDが優位
3. **好きだから買う（ブランド）** - 情緒的価値・世界観への共感

### 核心概念

- **流暢性**: 心にスムーズに入ってくる度合い。接触頻度に比例して上昇
- **想起集合**: 購買時に思い出される選択肢の集合
- **PoP（Point of Parity）**: 競合と同じ土俵に立つための要件
- **PoD（Point of Difference）**: 競合に勝つための独自性・差別性
- **ユースケース**: 利用理由×利用タイミングの組み合わせ
- **ブランド価値規定**: 企業・製品の「らしさ」を言語化した設計図

## 使用場面（When to Use）

- マーケティング戦略の策定時
- 新商品・サービスのポジショニング設計時
- 競合分析とPoP/PoD定義時
- ターゲティング戦略の見直し時
- コミュニケーション施策の設計時
- ブランド価値規定の策定・見直し時
- インサイト発掘のためのユーザー調査設計時
- リポジショニング検討時

## フレームワーク/プロセス

### Phase 1: ユースケースの網羅

```
[ユースケース×ポジショニング分析テンプレート]

| ユースケース | ポジショニング | 主たるユーザー像 |
|-------------|---------------|-----------------|
| 利用理由 | 利用タイミング | 競合 | PoP | PoD | ユーザープロファイル |
```

**実行ステップ:**
1. ブランドの利用理由を網羅的に書き出す
2. 各利用理由に対応する利用タイミングを特定
3. ユースケースごとに競合を設定（直感的に決めない）
4. 競合と比較されるためのPoP（必須要件）を定義
5. 競合に勝つためのPoD（差別性）を特定
6. 主たるユーザー像を補記

### Phase 2: ポジショニングの設計

```
[ポジショニング構造図]

競合: [誰と比較されるか]
  ↓
PoP（Point of Parity）: [同じ土俵に立つための要件]
  - 要件1
  - 要件2
  ↓
PoD（Point of Difference）: [独自性・差別性]
  - 強み1
  - 強み2
```

**重要原則:**
- 競合は顧客の利用目的によって異なる
- ショッピングミッション起点で競合を考える
- 直感的な競合設定は競争構造を見誤る原因

### Phase 3: ターゲティングの三層構造

```
[ターゲット階層モデル]

階層1: コンセプチュアルターゲット
  目的: ブランドコンセプト・商品仕様の決定
  定義: ブランドの理想のユーザー（ペルソナ）
  特徴: 価値観・好み・趣味が言語化された人物像

階層2: コミュニケーションターゲット
  目的: ブランドコミュニケーション（広告・広報）
  定義: 理想のユーザーが所属する準拠集団
  特徴: メディア接触・コンテンツ消費パターン

階層3: プロモーションターゲット
  目的: 営業・販促
  定義: 買ってくれる可能性がある人全員
  特徴: コミュニケーションターゲットより広いスパン
```

**「人より意図」の原則:**
- 人物像よりも「利用理由・意図」を単位にターゲティング
- 芋焼酎問題（特定ペルソナに絞ると他顧客に不具合）を回避
- コンセプチュアルターゲットの陳腐化を防止

### Phase 4: ブランド価値規定

```
[ブランド価値規定テンプレート]

■ 多面体構造（ユースケース×ポジショニング）
  面1: [ユースケース1] × [ポジショニング1]
  面2: [ユースケース2] × [ポジショニング2]
  ...

■ 基盤要素（全ユースケース共通）
  - ミッション/ビジョン/パーパス: [組織が成し遂げたいこと]
  - パーソナリティ: [ブランドが人だったらどんな性格か]
  - シンボル: [ロゴ・色・サウンドロゴ・タグライン]
  - Reason to Believe: [信頼に値する根拠・事実]
```

**ブランド形成の仕組み:**
1. ブランド価値規定（企業の決め事）が源泉
2. 一貫性を持ったコミュニケーションとして発露
3. 消費者の心の中で印象が蓄積
4. ブランドとして形成・強化

### Phase 5: インサイトの発掘

```
[購買理由マトリクス]

           | 一般性：高 | 一般性：低 |
自明さ：高 | 激戦ゾーン | 低優先度   |
           | (1)        | (3)        |
自明さ：低 | 成長ゾーン | ニッチ     |
           | (2)        | (4)        |

プライオリティ:
- リーダー: (1)→(2)→(4)→(3)
- チャレンジャー: (2)→(1)→(4)→(3)
- ニッチ: (4)→(1)→(3)
```

**インサイト = 購買理由 + 背後の心の機序**

| 購買理由 | 心の機序 |
|---------|---------|
| 止渇 | 生存の継続 |
| 気分を上げる | モチベーションの欠落 |
| 現実から目を背ける | 現実逃避 |
| ハメを外す | 変身願望・自己承認欲求 |

### Phase 6: コミュニケーション設計

```
[インパクト構造]

「びっくり」 → 「納得」

びっくりの型:
1. 異化: 見慣れた概念に通常と違う表現をあつらえる
2. 笑いの提供: ユーモアで感情の起伏を起こす
3. 個の物語: 具体的な個人への感情移入

メディア考慮事項:
- 接触態度（能動的 vs 受動的）
- 接触許容時間（TV:15秒 / 店頭:2秒）
```

## 具体的な実行手順

### Step 1: 現状分析（As-is）

1. 自社ブランドの現在の利用理由を列挙
2. 各利用理由に対する競合を特定
3. 現在のPoP/PoDを言語化
4. 消費者の認知・イメージを調査

### Step 2: ユースケース網羅

1. 利用理由×利用タイミングでユースケースを作成
2. 各ユースケースで競合設定
3. ユースケースごとにPoP/PoDを定義
4. プライオリティを設定

### Step 3: ターゲティング設計

1. 利用理由起点でターゲットを設定（人より意図）
2. コンセプチュアルターゲットの具体化
3. コミュニケーションターゲットの準拠集団特定
4. プロモーションターゲットのスパン設定

### Step 4: ブランド価値規定

1. 多面体の各面（ユースケース×ポジショニング）を定義
2. 共通基盤（ミッション・パーソナリティ・シンボル・RtB）を設定
3. ブランドコンセプト・機能的価値・情緒的価値を言語化

### Step 5: インサイト発掘

1. 顕在的購買理由の網羅（ユーザーインタビュー）
2. 潜在的購買理由の探索（観察・アブダクション）
3. 心の機序の言語化
4. 自明さ×一般性マトリクスでプライオリティ付け

### Step 6: コミュニケーション設計

1. 「びっくり」要素の設計（異化/笑い/個の物語）
2. 「納得」への導線設計
3. メディア特性に合わせた調整
4. 接触頻度（フリークエンシー）の設計

## 入力/出力

### 入力

- 自社製品・サービスの概要
- 市場・競合環境情報
- 顧客データ・購買データ
- 既存のブランド資産
- ビジネス目標・KPI

### 出力

1. **ユースケース×ポジショニング一覧表**
   - 全利用理由の網羅
   - 競合/PoP/PoD定義

2. **ターゲット三層設計書**
   - コンセプチュアルターゲット詳細
   - コミュニケーションターゲット定義
   - プロモーションターゲット範囲

3. **ブランド価値規定書**
   - 多面体構造図
   - 基盤要素（ミッション・パーソナリティ・シンボル・RtB）

4. **インサイトマップ**
   - 購買理由×心の機序マトリクス
   - プライオリティ付き施策方針

5. **コミュニケーション設計書**
   - びっくり→納得構造
   - メディアプラン

## 注意点

### やるべきこと

- **競合はユースケースごとに設定**: 直感的な競合設定は危険
- **人より意図でターゲティング**: ペルソナ固執は陳腐化リスク
- **具体から抽象へ**: ユーザーに感情移入し具体的利用シーンを想定
- **観察はスキャンするように**: 概念単位の認識を超えて解像度高く
- **フォーカスポイントを変えて観察**: 同じ風景から複数の意味を抽出
- **アブダクション活用**: 観察から共通性を探り根源的理由を発見
- **決め事は3つまで**: 社員が意識できる情報量に絞る
- **Routinizerをつくる**: 行動変換の仕組みを日常に埋め込む

### やってはいけないこと

- **フレームワークの機械的適用**: 自社特性に合わせた修正が必要
- **ポジショニングをマトリクスだけで考える**: 競合/PoP/PoDの3要素で言語化
- **インサイト=潜在的という神話**: 顕在的購買理由も同様に重要
- **インサイト=ネガティブ感情という神話**: 利他的・善意のインサイトも存在
- **いいたいことをいえば伝わる幻想**: びっくり→納得の構造が必要
- **決め事の乱立**: More is less（多すぎると何も伝わらない）
- **規定演技型のブランディング**: 自由演技で進化し続ける姿勢を

### 参照点と意思決定

人は意思決定の参照点を気分で恣意的に決める。この性質により：
- 提示の仕方で比較対象を柔軟に設定できる
- 直感的でない競合との比較検討も可能
- これがポジショニングが機能する理由

### ブランドと概念の構造

ブランドは概念と相似形の構造を持つ：
- 名前 + 特徴・意味 = 概念
- 時間とともに意味が拡張・進化
- 矛盾する要素も包含しながら発展

人は概念を単位に認識形成するからこそ、ブランディングが重要。

### アイデア創生

新しいアイデア = 概念の合成
- 2つの概念を足し算
- 特徴・意味を間引き
- 新しい概念を生み出す

例: 猫カフェ = 猫（かわいい・自由・気まぐれ）+ カフェ（空間・立地）
