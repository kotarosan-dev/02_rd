# 心理的リソース・マネジメント（Psychological Resource Management）

## 概要

「心理的リソース」とは「面倒くさいけど、やるぞ！」と奮起するための心のエネルギーである。このリソースは有限であり、消耗すれば思考力・判断力・自己制御力が低下し、チームの生産性と人間関係が悪化する。本フレームワークは、心理的リソースの「消耗要因の特定」「浪費の停止」「積極的な増幅」という3ステップでチームを活性化する。

## 使用場面（When to Use）

- チームの生産性が低下している
- メンバーの表情が暗い、雑談や笑顔が減っている
- ミスや手戻りが増えている
- メンバーが主体的に動かない、提案が出てこない
- 離職者やメンタル不調者が発生している
- リーダー自身が「どうすればいいかわからない」と感じている
- 「性善説」と「性悪説」の間で揺れ動いている

## 核心概念：有限説

従来のマネジメントは「性善説」（信じて任せれば自ら動く）と「性悪説」（見張らなければサボる）の二元論に陥りがちである。「有限説」はこれを超える第三の人間観である。

**有限説の原則**：
- 人間は本来「善」でも「悪」でもなく、ただ「有限」である
- 心理的リソースが満たされていれば「善」的に行動する
- 心理的リソースが枯渇すれば「悪」的になる
- マネジメントの要諦はリソースの状態を見極め、消耗を防ぎ、増幅させること

## フレームワーク：3つの消耗領域

心理的リソースは「アタマ・ココロ・カラダ」の3領域で消費される。

### アタマ（思考）の消耗要因

| カテゴリ | 具体例 |
|----------|--------|
| わからなさ | チームの目標・方針が不明確、自分の役割が曖昧、成功の定義が見えない |
| 複雑さ | 方針・優先順位が頻繁に変更、複数プロジェクトの同時進行 |
| 矛盾や衝突 | 言動の一貫性がない、相反する要求や指示、リソースの取り合い |

### ココロ（感情）の消耗要因

| カテゴリ | 具体例 |
|----------|--------|
| 不安・恐れ | 失敗への不安、否定・叱責への恐れ、評価の不公平感 |
| 怒り・不満 | 理不尽な要求、業務の非効率さ |
| 自己効力感の低下 | 行動や能力を否定される、成功体験を得られない、関心を向けられない |

### カラダ（身体）の消耗要因

| カテゴリ | 具体例 |
|----------|--------|
| 生活習慣 | 睡眠不足、偏った食事、運動不足 |
| 身体的負荷 | PC画面の長時間注視、温度・騒音、満員電車通勤 |
| 緊張状態 | 締切に追われる、クレーム対応、長時間労働 |

**重要**：3領域は密接に連動している。アタマの消耗→ココロの不安→カラダの緊張→睡眠不足→さらなるアタマの消耗、という悪循環が生じやすい。

## 消耗のサインチェックリスト

### 行動の変化
- [ ] ミスや手戻りが増えている
- [ ] 判断や決定が遅い
- [ ] ルールや締切が守られない
- [ ] 作業に時間がかかる
- [ ] 主体的に動かない
- [ ] 新しい提案が出てこない
- [ ] 遅刻や欠勤が多い

### コミュニケーションの変化
- [ ] 雑談や笑顔が減った
- [ ] 必要な情報共有がされない
- [ ] 相互支援が起こらない
- [ ] 他責的な態度が目立つ
- [ ] 会議での発言が少ない
- [ ] 責任の押し付け合いがある
- [ ] 違和感が放置されている

### 外見の変化
- [ ] ボーっとしている時間が多い
- [ ] 表情が固い
- [ ] 姿勢が悪い
- [ ] 声のトーンが下がる

## 実行プロセス

### Step 1: 消耗のサインを検知する

上記チェックリストでチームの状態を把握する。「なんとなく雰囲気が悪い」という段階で対処を開始することが重要。放置すれば負の連鎖が加速する。

### Step 2: 真因を特定する

表面的なサインの背後にある消耗要因を探る。

**質問例**：
- このメンバーは何に心理的リソースを費やしているか？
- アタマ・ココロ・カラダのどの領域で消耗しているか？
- リーダーである自分が消耗の原因になっていないか？
- 構造的な問題（役割の曖昧さ、情報の不足など）はないか？

### Step 3: 消耗を止める（バケツの穴を塞ぐ）

消耗要因を取り除く。増幅より先に消耗の停止が優先。

**アタマの消耗対策**：
- 明確な指示と優先順位の提示
- 判断基準やルールの明文化
- 情報の整理と可視化

**ココロの消耗対策**：
- 安全な相談環境の整備
- 成功体験を生む小さな目標設定
- 承認とフィードバックの提供

**カラダの消耗対策**：
- 業務量の適正化
- 休息の確保
- 物理的環境の改善

### Step 4: 心理的リソースを増幅する

消耗を止めた後、積極的にリソースを増やす。

**「願い」の発見と尊重**：
- メンバーの「本当の願い」を対話から理解する
- 願いが叶う、あるいは叶いそうだと信じられる環境を整える
- すべての願いを叶えられなくとも、尊重する姿勢を見せる

**チームの共通言語化**：
- 「心理的リソース」という概念をチームで共有する
- 「今日はリソースが低い」「この作業はリソース消費が大きい」と言い合える文化を醸成

## サイパ（心理的リソース効率）評価

施策や意思決定を「サイパ」の視点で評価する。

**サイパの定義**：
```
サイパ = 生み出される価値 / 消費される心理的リソース
```

**評価の問い**：
- この施策は心理的リソースをどれだけ消費するか？
- 消費に見合う価値を生み出しているか？
- より低いリソース消費で同等の価値を生む方法はないか？

**サイパの悪い例**：
- 目的の不明確な会議
- 重複する報告業務
- 頻繁な方針変更
- 曖昧な指示による手戻り

## リーダー自己チェック

リーダー自身が「心理的リソース泥棒」になっていないか確認する。

**チェック項目**：
- [ ] 不機嫌な表情をしていないか
- [ ] 指示が曖昧になっていないか
- [ ] メンバーが話しかけにくい雰囲気を出していないか
- [ ] 自分の期待と違う成果物に過剰に反応していないか
- [ ] メンバーの「願い」を無視していないか

**リーダーの心理的リソースも有限**：
- 自身のリソース状態をモニタリングする
- 消耗パターンを把握する
- 余裕を意識的に確保する

## 出力例

### チーム診断レポート

```
【現状評価】
消耗サイン検出数：7/18項目
主要消耗領域：ココロ（不安・恐れ）
推定原因：プロジェクトの方向性変更が頻発し、「何をすれば正解かわからない」状態

【優先対応】
1. 方針変更の背景と今後の見通しを丁寧に説明する場を設ける
2. 「迷ったらまず相談」のルールを明文化
3. 小さな成功体験を生むマイルストーンを設定

【サイパ改善案】
- 週次進捗会議を隔週に変更（リソース消費↓、価値は維持）
- 報告フォーマットの統一（思考の消耗↓）
```

## 注意点

- 心理的リソースは測定が難しい。サインの観察と対話から推測する
- 消耗の原因を特定する際、リーダー自身が原因である可能性を除外しない
- 「もっと頑張れ」という声かけは逆効果。リソースが枯渇した状態で行動変容を求めても成功しない
- メンバーの「願い」はそれぞれ異なる。一律の対応ではなく個別理解が必要
- 構造的な問題（役割・ルール・仕組みの不備）がある場合、個人への働きかけだけでは解決しない
