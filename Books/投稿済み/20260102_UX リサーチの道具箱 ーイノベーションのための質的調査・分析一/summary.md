## 1. 一言で言うと

**ユーザの「声」ではなく「体験」を聴き、ユーザ自身も気づいていない暗黙のニーズを発見するための7つの実践的な質的調査・分析手法を体系化した実務書。**

---

## 2. 主要なポイント

### ポイント1: ユーザの「声」ではなく「体験」を聴く
- ユーザの声（V）はユーザの体験（x）を自己分析した結果に過ぎない（V=f(x)）
- 「素人分析」の結果を改めて分析しても新たな発見は得られない
- アンケートやグルインではなく、「コンテクスチュアル・インクワイアリー」で生の体験を入手すべき

### ポイント2: 「師匠と弟子」のインタビュー手法
- ユーザが師匠、インタビューアが弟子という人間関係モデル
- 4ステップ: 教えを請う → 根掘り葉掘り → 確認する → フォーカスを移動する
- 「なぜ」ではなく「どのように」を聞く（意思決定プロセスを把握）

### ポイント3: ペルソナは「パターン」から作る
- ペルソナ = ①パターン化 + ②擬人化 + ③優先順位付け
- 複数ユーザの共通パターンを骨格として「合成人間」を創作する
- プライマリーペルソナ1人に絞り、要求の対立を解決する

### ポイント4: ジャーニーマップでユーザ体験を可視化
- 時間軸に沿って行動・思考・感情・ペインポイントをマッピング
- 「製品」だけでなく「体験全体」のスコープで描く（座席だけでなく旅全体）
- マルチタッチポイントにおけるインタラクションをデザイン

### ポイント5: ジョブ理論で本質的ニーズを把握
- 「人は片付けるべきジョブのために製品を雇う」
- 製品カテゴリを破壊し、新たな成長機会と脅威を発見
- 4つのフォース（押し・引き・惰性・不安）で顧客の意思決定を理解

## 3. 実践への示唆（明日から使える3つのアクション）

### アクション1: インタビューを「弟子入り」に変える
- 事前に用意した質問を読み上げるのではなく、ユーザの話の中から「質問を見つける」
- 「文脈をクリック」: ユーザの話に含まれるリンク箇所を見つけて深掘りする
- 話の糸口パターン: ①体験の有無 → ②体験の頻度 → ③直近の体験 の順で聞く

### アクション2: 付箋紙で質的データを分析する
- データから重要な箇所を抜き出し、20文字以内に要約してカード化
- KJ法の要領でグループ化 → 図解化 → ストーリーラインを再構築
- 分析結果は「結論」ではなく「仮説」。現場に戻って検証を繰り返す

### アクション3: バリュープロポジション・キャンバスを描く
- 右側（顧客プロフィール）: ジョブ・ペイン・ゲイン を書き出す
- 左側（価値マップ）: 製品・ペインリリーバー・ゲインクリエーター を対応させる
- 「顧客が望み、競合が提供できず、自社が提供できる」価値を見つける

## 5. こんな人におすすめ

| ターゲット | おすすめ理由 |
|-----------|-------------|
| **プロダクトマネージャー** | ユーザニーズの把握から製品企画への展開方法を体系的に学べる |
| **UXデザイナー・リサーチャー** | インタビュー、ペルソナ、ジャーニーマップなど実務で使う手法の正しい使い方を確認できる |
| **新規事業担当者** | ジョブ理論とバリュープロポジション・キャンバスで事業アイデアを検証できる |
| **エンジニア・開発者** | 「誰のために何を作るのか」を明確にし、無駄な機能開発を減らせる |
| **マーケター** | アンケート・グルインに頼らない質的調査の価値と手法を理解できる |

## 6. 書籍構成（7つ道具）

1. **ユーザ調査概論** - 調査と評価の違い、ハイテク人類学の潮流
2. **ユーザインタビュー** - 師匠と弟子モデル、文脈をクリック
3. **データ分析** - 質的データの編集、KJ法、パターン発見
4. **ペルソナ** - パターン化・擬人化・優先順位付け
5. **シナリオ** - AS-IS/TO-BE、ペルソナシナリオ法
6. **ジャーニーマップ** - タッチポイント、ビフォーアフター比較
7. **ジョブ理論** - ジョブ、プログレス、4つのフォース
8. **キャンバス** - ビジネスモデル・キャンバス、VP・キャンバス
