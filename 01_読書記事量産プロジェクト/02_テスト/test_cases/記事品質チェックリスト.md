# 記事品質チェックリスト

このチェックリストは、`book-article-generator` skillで生成された記事が公開品質に達しているかを確認するためのものです。

## 構造チェック（必須 - 全項目クリア必須）

- [ ] **タイトル**: 30文字以内で、読者の悩みが明確
- [ ] **導入部**: 「こんな問いを持っていませんか?」があり、3つの箇条書き
- [ ] **書誌情報**: 書名と著者が正しく明記されている
- [ ] **3つの問い**: 「Why → How → Future」の階層構造になっている
- [ ] **処方箋の根拠**: 著者の信頼性・独自性・実用性が説明されている
- [ ] **読了後の未来**: 具体的な変化や成長が描かれている
- [ ] **まとめ**: 簡潔で、読者の背中を押す内容

## 内容チェック（品質基準）

- [ ] **ネタバレ回避**: 解決策の詳細を書きすぎず、期待感を残している
- [ ] **具体性**: 読者の悩みが具体的で「自分ごと化」できる
- [ ] **信頼性**: 著者の実績や本の独自性が明確に示されている
- [ ] **文字数**: 1500-2000文字の範囲内
- [ ] **問い駆動**: 単なる書籍紹介ではなく、読者の悩み起点になっている

## 文体チェック（読みやすさ）

- [ ] **敬体統一**: 「です・ます」調で統一されている
- [ ] **平易な表現**: 専門用語を避け、ビジネスパーソンに馴染みやすい
- [ ] **対話的**: 読者を「あなた」と呼びかけている
- [ ] **親しみやすさ**: 硬すぎず、note読者に適した語り口

## ターゲット読者チェック（明確性）

- [ ] **ペルソナ設定**: 具体的なターゲット読者が設定されている（例: 営業に悩む20代）
- [ ] **共感ポイント**: 読者が「これは自分のための記事だ」と感じられる
- [ ] **背中を押す**: 最後に行動を促す一言がある

## Quick Reference（Skill標準チェック）

- [ ] 問い駆動型か
- [ ] 3つの問いがWhy→How→Futureか
- [ ] 処方箋の根拠が3観点を含むか
- [ ] ネタバレ回避
- [ ] 7セクション構造厳守
- [ ] 文字数1500-2000文字
- [ ] 文体が親しみやすいか
- [ ] ターゲット読者が具体的か
- [ ] 共感ポイントがあるか
- [ ] 背中を押す一言があるか

## 総合評価

### 公開可否判断

**公開OK条件**:
- 構造チェック全7項目がクリア
- 内容チェック5項目中4項目以上がクリア
- 文体チェック4項目全てがクリア
- Quick Reference 10項目中9項目以上がクリア

**要修正**:
- 上記条件を満たさない場合、該当箇所を修正して再チェック

**備考欄**:
- 気になる点や改善提案を記入
- 次回記事作成時の参考にする
