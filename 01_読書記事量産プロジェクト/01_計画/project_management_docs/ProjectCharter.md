# プロジェクト憲章：読書記事量産プロジェクト

## プロジェクト目的
書籍データから高品質なnote記事を効率的に生成し、継続的に公開する体制を構築する。

## ビジネス価値
- note読書アカウントの差別化と成長
- 書籍知識のアウトプット習慣化
- AIとの協業による生産性向上

## スコープ

### 含まれるもの
- Claude Code Skillの開発
- 記事フォーマットの標準化
- サンプル記事の作成
- 運用ワークフローの確立

### 含まれないもの
- note外のプラットフォームへの展開
- 動画コンテンツの作成
- 書籍以外（論文、記事など）のコンテンツ化

## 成功基準
1. Skillを使って15分以内に記事下書きを生成できる
2. 月間5本以上の記事を安定して公開できる
3. 記事の品質が一定水準（7セクション構造、1500-2000文字）を維持

## ステークホルダー
- **プロジェクトオーナー**: あなた（記事企画・最終編集・公開）
- **技術支援**: Claude Code（記事生成・構造化）
- **読者**: note読書アカウントのフォロワー（20-40代ビジネスパーソン）

## リスクと対策

| リスク | 影響 | 対策 |
|:---|:---|:---|
| 記事の質がAI頼りで低下する | 読者離れ、信頼性低下 | 品質チェックリスト厳守、人間編集必須 |
| 継続が困難になる | プロジェクト停滞 | 週1本から開始、バッチ処理で効率化 |
| Books/の書籍データが不足する | ネタ切れ | 既存8冊で16-24本可能、段階的に追加 |
| Skillの保守が困難になる | 使いにくくなる | コメント充実、変更履歴記録 |

## 成功指標（KPI）

### 定量指標
- 記事生成効率: 45分以内/1記事
- 公開頻度: フェーズ1（週1本） → フェーズ2（週2-3本） → フェーズ3（週5本）
- 書籍カバー率: 80%を6ヶ月以内

### 定性指標
- 品質チェックリスト全項目クリア
- note「スキ」平均10以上
- 「誇れる記事」月3本以上
