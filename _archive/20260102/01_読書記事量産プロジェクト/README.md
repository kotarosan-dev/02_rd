# 読書記事量産プロジェクト

## 概要
書籍データ（Books/フォルダ）から、note投稿用の読書記事を体系的に生成するプロジェクト。
Claude Code Skill `book-article-generator` を活用し、「問い駆動型」の独自フォーマットで記事を量産します。

## コンセプト
**あなたの「問い」に答える一冊を処方する、本の相談室**

単なる書評ではなく、読者の具体的な悩み（問い）を起点に、解決策として本を「処方箋」のように提示します。

## プロジェクト構造
```
01_計画/          - 記事フォーマット仕様、Skill仕様、ワークフロー図
02_テスト/        - サンプル記事、品質チェックリスト
03_実装/          - 公開済み記事、下書き、テンプレート
04_その他/        - 作業メモ、参考資料
```

## 主要成果物
- [x] Claude Code Skill: `book-article-generator`
- [ ] サンプル記事3本（無敗営業、SPIN、ハーバード）
- [ ] 記事執筆テンプレート
- [ ] 書籍リスト管理シート

## 使い方

### 基本ワークフロー（単一記事生成 - 45分）
1. **企画（5分 - 人間）**: Books/から記事化する書籍を選定
2. **書籍分析（10分 - AI）**: Claude Codeで `book-article-generator` skillを起動、フェーズ1実行
3. **記事構成設計（10分 - AI+人間）**: フェーズ2実行、確認・微調整
4. **記事執筆（10分 - AI）**: フェーズ3実行
5. **最終編集（15分 - 人間）**: 体験追加、文体調整
6. **公開・アーカイブ（5分 - 人間）**: noteに投稿、`03_実装/published_articles/`に保存

### バッチ生成ワークフロー（週末モデル）
- 月-金: 企画（書籍選定、問い決定）
- 土曜午前: AI一括生成（3-5本の下書き）
- 土曜午後: 人間編集（品質チェック、加筆）
- 日曜: 公開準備（優先順位付け、順次公開）

**期待効果**: 月20本公開ペースが可能
